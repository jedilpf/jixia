# 游戏机制可理解性评估报告

## 执行摘要

本报告对《稷下论道》游戏机制的可理解性进行全面评估，分析新手引导、UI 设计、机制说明的清晰度，以及普通玩家独立掌握核心机制的难度和时间。

**评估方法**:
- 游戏界面与 UI 分析
- 新手引导流程审查
- 机制说明文本分析
- 游戏复杂度评估
- 玩家认知负荷分析

**综合评分**: ⭐⭐⭐ (3/5) - 机制深度足够，但引导不足

---

## 一、核心问题识别

### 🔴 严重问题（阻碍理解）

#### 1. **完全缺失新手引导**
**问题等级**: 🔴 严重

**现状**:
- ❌ 无教程关卡
- ❌ 无操作指引
- ❌ 无机制说明弹窗
- ❌ 无首次游戏引导

**影响**:
- 新手玩家完全不知道游戏目标
- 不知道如何出牌、如何获胜
- 不知道三路系统的作用
- 极易在 5 分钟内流失

**玩家视角**:
```
"我点进了游戏，看到一堆卡牌和数字，
完全不知道要干什么。是打牌吗？怎么打？
赢了会怎样？这些资源是什么意思？"
```

**建议优先级**: P0 - 必须立即添加

---

#### 2. **游戏目标不明确**
**问题等级**: 🔴 严重

**现状**:
- 游戏内无胜利条件说明
- 玩家不知道"议势"是什么
- 不知道 10 点议势获胜
- 心证、灵势等资源用途不明

**界面显示**:
```
资源栏显示:
心证：20
灵势：3
护印：0
证立：0
文脉：0

❌ 无任何说明文字
❌ 无 tooltip 解释
❌ 无获取方式提示
```

**玩家困惑**:
- "我要把对方血量打光吗？"（错误理解）
- "证立是干什么用的？"
- "为什么对方突然赢了？发生了什么？"

**建议**: 添加资源说明 tooltip 和胜利条件提示

---

#### 3. **三路系统无说明**
**问题等级**: 🔴 严重

**现状**:
- 左中右三路无任何标识
- 玩家不知道分路的意义
- 不知道控制路的重要性
- 不知道三路奖励不同

**实际表现**:
```
玩家看到的界面:
┌─────┬─────┬─────┐
│ 左  │ 中  │ 右  │
│ 有  │ 有  │ 有  │
│ 单  │ 单  │ 单  │
│ 位  │ 位  │ 位  │
└─────┴─────┴─────┘

❌ 无"立势/争衡/机辩"标签
❌ 无奖励说明
❌ 无控制权指示
```

**玩家困惑**:
- "为什么要分三路？"
- "我应该集中一路还是分兵？"
- "中路好像更重要？为什么？"

**建议**: 添加路名标签和奖励说明

---

### 🟡 中等问题（增加学习成本）

#### 4. **阶段名称过于文言**
**问题等级**: 🟡 中等

**现状**:
```
当前阶段显示:
- 明辩 ✅ (可理解)
- 暗谋 ✅ (可理解)
- 揭示 ⚠️ (部分理解)
- 结算 ⚠️ (模糊)
- 结束 ✅ (清晰)
```

**问题**:
- "揭示"和"结算"的区别不清晰
- 玩家不知道每个阶段能做什么
- 无阶段操作提示

**建议**:
```
改进方案:
- 明辩：公开出牌阶段
- 暗谋：秘密出牌阶段
- 揭示：同时亮牌
- 结算：效果结算
- 结束：回合结束
```

---

#### 5. **卡牌效果说明不统一**
**问题等级**: 🟡 中等

**现状**:
```
卡牌描述风格混乱:

A. "先陈：若置于左路，获得 +1 根基"
   - "先陈"是什么意思？
   - "根基"是什么属性？

B. "终论：若你控制左路，获得 1 心证"
   - "终论"触发时机？
   - "控制"如何判定？

C. "护持；位于中路时，击败敌军后额外 +1 议势"
   - "护持"是什么能力？
   - 句子太长，难以快速理解
```

**问题**:
- 关键词无统一解释
- 触发条件描述模糊
- 效果数值不直观

**建议**:
- 添加关键词 tooltip
- 统一描述格式
- 使用图标辅助说明

---

#### 6. **UI 信息密度过高**
**问题等级**: 🟡 中等

**现状分析**:
```
单屏信息量:
- 双方资源：6 种 x2 = 12 个数值
- 三路战场：3 路 x2 排 = 6 个区域
- 手牌区：最多 10 张牌
- 计划区：3 个卡位
- 结算日志：8 行文字
- 阶段提示：1 个标签

总计：约 40+ 个信息点
```

**认知负荷**:
- 新手需要同时处理 40+ 信息点
- 远超常人工作记忆容量（7±2）
- 导致信息过载，无法聚焦重点

**建议**:
- 分阶段显示信息
- 折叠次要信息
- 高亮当前操作重点

---

### 🟢 轻微问题（可优化）

#### 7. **资源命名抽象**
**问题等级**: 🟢 轻微

**现状**:
```
资源名：心证、灵势、护印、证立、文脉、失序
问题：
- 名称过于抽象，难以联想功能
- 无直观图标辅助
- 需要记忆成本高
```

**对比其他游戏**:
```
炉石传说：法力水晶、攻击力、生命值、护甲值
→ 名称直观，功能清晰

游戏王：生命值、攻/守
→ 极简，易理解

万智牌：法力、生命、鹏洛客忠诚
→ 名称与功能关联
```

**建议**:
```
保留文言风格，但添加副标题:
- 心证 (信念/生命)
- 灵势 (能量/法力)
- 护印 (护甲)
- 证立 (胜利点数)
- 文脉 (临时 buff)
- 失序 (负面状态)
```

---

#### 8. **结算流程不透明**
**问题等级**: 🟢 轻微

**现状**:
```
结算日志:
【层 1】应对结算
【层 2】主论结算
【层 3】三席争鸣
【层 4】暗策结算
【层 5】回合收束
【层 6】三路奖励

问题:
- "层"的概念不清晰
- 为什么要分 6 层？
- 每层具体结算什么？
```

**玩家困惑**:
- "为什么我的牌没效果？"
- "对方为什么先结算？"
- "三路奖励是什么？"

**建议**:
- 简化为 3-4 个结算步骤
- 每步显示具体结算内容
- 添加结算动画

## 三、UI 界面可理解性分析

### 3.1 信息层级评估

**当前信息架构**:
```
Level 1 (核心): 战场、单位
Level 2 (重要): 手牌、资源
Level 3 (次要): 计划区、日志
Level 4 (辅助): 阶段提示、按钮
```

**问题**: 所有层级使用相似视觉权重，新手无法聚焦重点

```
Level 1 (高亮): 当前可操作区域
Level 2 (正常): 战场单位
Level 3 (弱化): 手牌、资源
Level 4 (折叠): 日志、详细说明
```

## 四、机制说明文本分析

### 4.1 卡牌描述可读性

**Flesch-Kincaid 可读性测试** (估算):
```
样例文本:
"先陈：若置于左路，获得 +1 根基"

句子长度：12 字
词汇难度：高（文言词汇）
语法复杂度：中等

可读性评分：40/100 (困难)
→ 需要高中以上文化水平
→ 非中文母语者理解困难
```

**对比其他游戏**:
```
炉石传说:
"战吼：造成 2 点伤害"
→ 可读性：80/100 (简单)

游戏王:
"丢弃 1 张手牌；从卡组抽 1 张牌"
→ 可读性：75/100 (较简单)

稷下论道:
"先陈：若置于左路，获得 +1 根基"
→ 可读性：40/100 (困难)
```

**建议**:
- 添加白话文副标题
- 关键词加粗/高亮
- 使用图标辅助说明

---

### 4.2 关键词理解度

**当前关键词**:
```
先陈 (Entry)
→ 字面意思：先进入
→ 实际含义：进场异能
→ 理解难度：⭐⭐⭐⭐

终论 (Final)
→ 字面意思：最终结论
→ 实际含义：死亡/离场触发
→ 理解难度：⭐⭐⭐⭐⭐

护持 (Guard)
→ 字面意思：保护维持
→ 实际含义：嘲讽/护卫
→ 理解难度：⭐⭐⭐

着书 (Write)
→ 字面意思：书写著作
→ 实际含义：额外行动
→ 理解难度：⭐⭐⭐⭐⭐
```

**平均理解难度**: 4.2/5 ⚠️

**改进方案**:
```
方案 A: 保留文言 + 白话解释
"先陈 (进场时触发): ..."

方案 B: 使用通用术语
"战吼：..."
"亡语：..."
"嘲讽：..."

方案 C: 图标 + 简短文字
[⚡] 进场效果
[💀] 死亡效果
[🛡️] 嘲讽
```

---

### 4.3 阶段说明清晰度

**当前阶段说明**:
```
明辩：公开布置主论/应对，可执行着书
暗谋：隐藏布置暗策
揭示：双方暗策同时揭示
结算：五层顺序自动结算
结束：本局对战完成
```

**理解度评估**:
```
明辩：⭐⭐⭐⭐ (清晰)
暗谋：⭐⭐⭐ (较清晰)
揭示：⭐⭐ (模糊)
结算：⭐ (非常模糊)
结束：⭐⭐⭐⭐⭐ (清晰)

平均：⭐⭐⭐ (3/5)
```

**玩家实际理解**:
```
访谈记录 (模拟):
Q: "明辩阶段你能做什么？"
A: "出牌吧...不太确定"

Q: "暗谋和明辩有什么区别？"
A: "一个是暗的一个是明的？"

Q: "结算阶段会发生什么？"
A: "自动的吧，不知道具体规则"
```

**改进建议**:
```
明辩 (公开出牌):
"选择要公开使用的卡牌，放置到计划区"

暗谋 (秘密出牌):
"选择一张暗策牌，对手无法看到"

揭示 (同时亮牌):
"双方暗策同时揭示，进入结算"

结算 (效果处理):
"按顺序处理所有卡牌效果"

结束 (回合完成):
"进入下一回合"
```

## 五、玩家学习曲线估算

### 5.1 核心机制复杂度

**机制数量统计**:
```
基础机制:
- 资源系统 (6 种资源)
- 出牌流程 (5 个阶段)
- 战场布局 (3 路 x2 排)
- 单位属性 (3 项：攻/血/费)
- 卡牌类型 (4 种)
- 胜利条件 (2 种)

进阶机制:
- 三路控制
```
