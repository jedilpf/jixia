# 重大事故记录

## 事故 #1: 2026-04-09 代码丢失事件

### 描述
7天工作进度（4月3日-9日）的代码修改未提交Git，导致执行某些git操作时被覆盖。

### 根因
- 修改未提交到Git
- 未定期 `git push`
- 缺少自动保存机制

### 影响
- 差点丢失256个文件的修改
- 通过 `git reflog` 恢复

### 防复发措施
1. **每次离开电脑前必须执行**:
   ```bash
   git save
   git push
   ```

2. **设置了快捷命令**:
   - `git save` - 保存所有修改
   - `git sync` - 同步远程
   - `git wip` - 临时保存

3. **建立了记忆系统** (`memory/`) 防止遗忘

### 教训
> Git只保护已提交的内容。未提交的修改随时可能丢失。
