## 二、Agent角色定义

### 2.1 项目指挥官 (Commander)

**职责**：
- 接收用户需求，分解为具体任务
- 分配任务给合适的子Agent
- 监控任务进度，协调资源
- 最终质量审核

**技能**：
- 任务分解与分配
- 多Agent协调
- 进度跟踪与报告

**触发条件**：用户提出任何开发需求

---

### 2.2 架构师 (Architect)

| 子角色 | 职责 | 技能 |
|--------|------|------|
| 系统架构师 | 整体系统设计、模块划分 | 系统设计、技术选型 |
| 数据架构师 | 数据模型、数据库设计 | 数据建模、SQL优化 |
| 安全架构师 | 安全方案、权限设计 | 安全审计、漏洞防护 |

**工作产出**：
- 架构设计文档 (ADR)
- 技术选型方案
- 数据模型设计

---

### 2.3 策划 (Designer)

| 子角色 | 职责 | 技能 |
|--------|------|------|
| 剧情策划 | 故事脚本、角色设定 | 剧情写作、分支设计 |
| 数值策划 | 卡牌数值、经济系统 | 数值平衡、概率设计 |
| 关卡策划 | 辩题设计、章节规划 | 关卡设计、难度曲线 |
| 系统策划 | 玩法规则、系统设计 | 系统设计、文档撰写 |

**工作产出**：
- 剧情脚本 (StoryNode)
- 数值表 (Excel/JSON)
- 系统设计文档

---

### 2.4 开发者 (Developer)

| 子角色 | 职责 | 技术栈 |
|--------|------|--------|
| 前端开发 | UI组件、页面交互 | React, TypeScript, Tailwind |
| 后端开发 | API、数据库、服务 | Node.js, SQLite, WebSocket |
| 引擎开发 | 游戏逻辑、状态机 | TypeScript, 状态管理 |
| 工具开发 | 编辑器、脚本工具 | Node.js, CLI |

**工作产出**：
- 源代码 (.ts, .tsx)
- 单元测试
- 技术文档

---

### 2.5 美术 (Artist)

| 子角色 | 职责 | 技能 |
|--------|------|------|
| UI设计师 | 界面设计、交互原型 | Figma, CSS |
| 角色设计 | 角色立绘、表情 | AI绘图, 角色设计 |
| 场景设计 | 背景图、氛围图 | AI绘图, 场景设计 |
| 动效设计 | UI动效、转场 | CSS动画, Lottie |

**工作产出**：
- UI设计稿 (.pen)
- 图片资源 (.png, .jpg)
- 动效文件

---

### 2.6 测试 (QA)

| 子角色 | 职责 | 技能 |
|--------|------|------|
| 功能测试 | 功能验证、回归测试 | 测试用例设计 |
| 自动化测试 | E2E测试、CI集成 | Playwright, Jest |
| 性能测试 | 性能分析、优化建议 | Lighthouse, Profiler |

**工作产出**：
- 测试用例
- Bug报告
- 测试报告

---

### 2.7 运维 (DevOps)

| 子角色 | 职责 | 技能 |
|--------|------|------|
| 部署工程师 | 发布流程、环境管理 | Docker, CI/CD |
| 监控工程师 | 日志收集、告警配置 | Prometheus, Grafana |
| 数据工程师 | 数据备份、迁移 | SQL, 数据管道 |

**工作产出**：
- 部署脚本
- 监控配置
- 运维文档

## 三、工作流程

### 3.1 需求处理流程

```
用户需求
    │
    ▼
┌─────────────────┐
│  Commander 接收  │
│  分析需求类型    │
└────────┬────────┘
         │
    ┌────┴────┬────────────┬────────────┐
    ▼         ▼            ▼            ▼
  新功能    Bug修复      优化        文档
    │         │            │            │
    ▼         ▼            ▼            ▼
┌───────┐ ┌───────┐  ┌───────┐  ┌───────┐
│策划+  │ │ QA+   │  │开发+  │  │相关   │
│架构+  │ │开发   │  │QA     │  │角色   │
│开发   │ │       │  │        │  │       │
└───────┘ └───────┘  └───────┘  └───────┘
    │         │            │            │
    └─────────┴────────────┴────────────┘
                      │
                      ▼
              ┌───────────────┐
              │ Commander 审核 │
              │ 质量检查       │
              └───────────────┘
                      │
                      ▼
                  完成/反馈
```

### 3.2 任务优先级

| 优先级 | 标识 | 响应时间 | 适用场景 |
|--------|------|----------|----------|
| P0 | 🔴 紧急 | 立即 | 生产故障、阻断性Bug |
| P1 | 🟠 高 | 1小时内 | 核心功能问题 |
| P2 | 🟡 中 | 当天 | 常规功能开发 |
| P3 | 🟢 低 | 本周 | 优化、文档 |

## 五、知识库管理

### 5.1 文档结构

```
docs/
├── AI_TEAM.md              # 团队架构（本文件）
├── PROJECT_MEMORY.md       # 项目记忆
├── MEMORY.md              # AI交互记录
├── GAME_RULES.md          # 游戏规则
├── story/                 # 剧情文档
│   ├── STORY_WORLD_SETTING_v1.md
│   ├── STORY_CHARACTERS_v1.md
│   └── STORY_CHAPTERS_v1.md
├── battle/                # 战斗文档
│   ├── battle-design-索引.md
│   └── battle-spec-同步对战.md
└── dev/                   # 开发文档
    ├── dev-guide-开发者指南.md
    └── dev-spec-技术架构.md
```

### 5.2 知识更新规则

1. **每次任务完成后**：更新相关文档
2. **重大决策后**：创建ADR记录
3. **发现问题时**：记录到已知问题
4. **学到的经验**：更新最佳实践

## 六、质量标准

### 6.1 代码质量

- TypeScript严格模式
- ESLint规则通过
- 单元测试覆盖率 > 60%
- 无console.log残留

### 6.2 文档质量

- 中文表述清晰
- 代码示例完整
- 更新日期标注

### 6.3 策划质量

- 数值平衡
- 剧情无逻辑漏洞
- 分支完整

## 七、紧急响应

### 7.1 生产故障

```
故障发生
    │
    ▼
Commander 评估严重程度
    │
    ├── P0: 立即拉取 Developer + DevOps
    │
    └── P1: 排入紧急队列
```

### 7.2 数据丢失

```
DevOps 执行恢复流程
    │
    ├── 检查备份状态
    ├── 选择恢复点
    ├── 执行恢复
    └── 验证数据完整性
```

## 八、工具链

| 工具 | 用途 | 使用者 |
|------|------|--------|
| Git | 版本控制 | 全体 |
| npm | 包管理 | 开发 |
| TypeScript | 开发语言 | 开发 |
| React | 前端框架 | 前端开发 |
| Vite | 构建工具 | 前端开发 |
| Jest | 单元测试 | 开发+QA |
| ESLint | 代码检查 | 开发+QA |
| Figma/Pen | UI设计 | 美术 |

### Agent Prompt模板

所有Agent的Prompt模板位于 `ai/agents/` 目录：

| Agent | 模板文件 | 适用场景 |
|-------|----------|----------|
| Commander | `commander.md` | 任务分解、协调、验收 |
| Architect | `architect.md` | 系统设计、模块划分 |
| Designer | `designer.md` | 剧情、数值、关卡设计 |
| Developer | `developer.md` | 前端、后端、引擎开发 |
| Artist | `artist.md` | UI设计、视觉设计 |
| QA | `qa.md` | 功能测试、自动化测试 |
| DevOps | `devops.md` | 部署、监控、备份 |

使用方式：
1. 选择合适的Agent模板
2. 模板内容 + 任务详情 → 发送给AI工具
3. AI工具按模板定义的角色执行任务
