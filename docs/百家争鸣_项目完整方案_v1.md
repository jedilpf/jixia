## 一、项目背景

### 1.1 项目定位

《百家争鸣》是一款以**论场匹配、明辩暗谋、三席布阵、着书养势**为核心的同步博弈卡牌对战游戏。

**核心差异化**：
- 非传统轮流出牌，而是同步操作、统一揭示
- 公开博弈（明辩）与隐藏博弈（暗谋）结合
- 三席双层站位让"布阵"成为实战资源
- 着书机制让前中后期节奏分层明确

### 1.2 目标用户

| 用户类型 | 特征 | 核心诉求 |
|---------|------|---------|
| 策略玩家 | 喜欢深度思考、预判对手 | 信息博弈、策略深度 |
| 收集玩家 | 喜欢构筑、收集卡牌 | 卡牌多样性、构筑空间 |
| 休闲玩家 | 碎片时间、轻度体验 | 快速上手、短局时 |
| 竞技玩家 | 追求排名、天梯 | 公平竞技、平衡环境 |

### 1.3 市场定位

- **品类**：策略卡牌对战
- **对标产品**：炉石传说、杀戮尖塔、昆特牌
- **差异化优势**：同步博弈机制、论场分流生态

## 二、核心功能

### 2.1 战斗系统（核心）

#### 2.1.1 回合阶段流转

```
回合开始 → 明辩(10s) → 暗谋(10s) → 揭示 → 五层结算 → 回合结束
```

| 阶段 | 时长 | 核心操作 | 信息可见性 |
|------|------|----------|-----------|
| 回合开始 | - | 抽牌、资源恢复、持续效果 | 完全公开 |
| 明辩 | 10秒 | 主论、应对、着书 | 双方互见操作结果 |
| 暗谋 | 10秒 | 暗策放置 | 仅见"是否放置" |
| 揭示 | - | 翻开暗策 | 完全公开 |
| 结算 | - | 五层自动结算 | 完全公开 |

#### 2.1.2 五层结算顺序

1. **应对结算**：反诘、减伤、改目标、限制
2. **主论结算**：立论生效、门客登场、玄章进场
3. **三席争鸣**：先声席 → 主辩席 → 余论席（每席：前席 → 后席 → 主辩者）
4. **暗策结算**：反转、补刀、资源反赚、时序干扰
5. **回合结束**：状态衰减、胜负判定

│  │前席│后席│   │  │前席│后席│   │   │前席│后席│      │

**伤害传递规则**：前席 → 后席 → 主辩者

### 2.2 资源系统

| 资源 | 作用 | 获取方式 |
|------|------|---------|
| 心证 | 主辩者生命值，归零即败 | 初始值、卡牌效果 |
| 灵势 | 回合费用，用于打牌 | 每回合恢复+成长 |
| 护印 | 优先承伤保护 | 卡牌效果、论场被动 |
| 证立 | 短时强化 | 卡牌效果 |
| 失序 | 负面资源 | 特定卡牌代价 |
| 文脉 | 长期成长 | 着书积累 |

### 2.3 卡牌系统

#### 2.3.1 五大牌类

| 牌类 | 定位 | 常见作用 |
|------|------|---------|
| 立论 | 正面推进 | 伤害、护印、证立、施压 |
| 策术 | 操作调度 | 抽牌、换牌、资源调整 |
| 反诘 | 应对反制 | 减伤、反制、改目标 |
| 门客 | 战场单位 | 抢席、守席、持续输出 |
| 玄章 | 持续规则 | 常驻增益、体系支撑 |

#### 2.3.2 卡牌数量规划

- **首发**：160张设定集
- **首赛季**：+20张扩展
- **后续赛季**：每赛季+15~30张

### 2.4 论场系统

#### 2.4.1 首发四论场

| 论场 | 定位 | 核心效果 | 适配门派 |
|------|------|----------|---------|
| 稷下学宫 | 标准环境 | 首次立论获证立 | 全门派通用 |
| 火德论坛 | 进攻环境 | 前席争鸣+1辩锋 | 九阵堂、衡戒廷 |
| 藏书秘阁 | 成长环境 | 着书额外护印 | 礼心殿、万农坊 |
| 玄机观星台 | 信息环境 | 可见暗策牌类 | 星玄院、名相府 |

#### 2.4.2 论场结构

每个论场仅含三层效果：
1. **基础被动**：开局自动生效
2. **偏向效果**：特定条件触发
3. **论场技**：本局一次主动技能

### 2.5 着书机制

**定义**：每回合明辩阶段可将1张手牌放入着书区，转化为长期成长资源。

**设计意义**：
- 制造"眼前少赚 vs 后期兑现"的决策拉扯
- 形成清晰的前期-中期-后期节奏分层
- 让"弃牌"变成有价值的投资行为

**验收标准**：
- 60%+对局中主动使用着书
- 玩家能感知"后续变强"

## 三、技术架构

### 3.1 架构原则

1. **服务端权威**：客户端只做表现，胜负由服务端结算
2. **阶段白名单**：每个阶段只允许特定操作，非法操作拒绝
3. **可重放性**：同种子同输入得到同结果
4. **可追溯性**：每层结算均有日志，异常可定位

## 五、潜在风险及应对策略

### 5.1 核心风险矩阵

| 风险 | 概率 | 影响 | 等级 |
|------|------|------|------|
| 暗谋阶段沦为形式化操作 | 高 | 高 | 🔴 严重 |
| 三席机制感知弱 | 中 | 高 | 🟠 重要 |
| 着书成长收益感不足 | 中 | 高 | 🟠 重要 |
| 多系统并行导致实现割裂 | 中 | 中 | 🟡 一般 |
| 平衡维护成本过快上升 | 低 | 高 | 🟡 一般 |

### 5.2 风险应对策略

#### 风险1：暗谋阶段沦为形式化操作

**现象**：玩家只在暗谋阶段放一张小牌或空过，失去博弈意义。

**根因分析**：
- 暗策牌效果不够强力
- 暗策触发条件过于苛刻
- 公开信息不足以支撑"读牌"

**应对措施**：
1. 确保暗策至少覆盖四类高价值效果：反转、补刀、资源反赚、时序干扰
2. 设计"读公开信息→猜暗策"的明确反馈链路
3. 玄机观星台论场强化暗策信息可见性
4. 设置暗谋阶段有效提交率作为硬KPI

#### 风险2：三席机制感知弱

**现象**：玩家感受不到站位对胜负的影响，三席变成摆设。

**根因分析**：
- 前后席伤害传递规则不清晰
- 席位差异化不足
- UI表现不够突出

**应对措施**：
1. 强化伤害传递的视觉反馈（穿透动画、伤害数字）
2. 火德论坛论场强化前席争鸣价值
3. 设计席位专属卡牌（如"只能放入先声席"）
4. 建立三席数据看板（席位胜率、击破率、穿透率）

#### 风险3：着书成长收益感不足

**现象**：玩家感受不到着书带来的后续优势，机制失效。

**根因分析**：
- 文脉成长曲线过于平缓
- 着书联动卡不足
- 兑现时机不明确

**应对措施**：
1. 藏书秘阁论场强化着书收益
2. 设计明确的"着书→兑现"联动卡
3. 文脉成长可视化（灵势曲线图）
4. 着书相关卡牌优先开发

#### 风险4：多系统并行导致实现割裂

**现象**：规则变更与实现不同步，配置和代码双源分裂。

**根因分析**：
- 文档与代码更新不同步
- 缺乏统一的规则源
- 测试覆盖不足

**应对措施**：
1. 单一规则源文档作为准入门槛
2. 每周规则-实现对齐评审
3. 回放一致性测试纳入发布门禁
4. 配置变更必须有版本号和校验器

#### 风险5：平衡维护成本过快上升

**现象**：卡牌数量增加后，平衡调整变得困难。

**根因分析**：
- 卡牌设计缺乏统一框架
- 缺乏数据驱动的平衡机制
- 热修能力不足

**应对措施**：
1. 建立固定平衡节奏（每周例会）
2. 关键指标面板（胜率、使用率、对局时长）
3. 配置热更新能力
4. 回滚机制

### 5.3 风险监控指标

| 指标 | 目标值 | 监控频率 |
|------|--------|---------|
| 暗谋有效提交率 | >70% | 每日 |
| 着书使用率 | >60% | 每日 |
| 三席穿透率 | 15-25% | 每周 |
| 对局时长中位数 | 3-4分钟 | 每日 |
| 论场分流比 | 各论场>15% | 每周 |

## 七、验收标准总览

### 7.1 产品体验验收

- [ ] 每局平均时长稳定在3~4分钟
- [ ] 前2回合"可行动感"明显
- [ ] 三席与前后席对胜负有决定性影响
- [ ] 着书使用率>60%，且玩家感知"后续变强"
- [ ] 暗谋阶段有真实博弈价值
- [ ] 新玩家3局内理解明辩/暗谋/着书

### 7.2 技术质量验收

- [ ] 对战逻辑完全可重放
- [ ] 同步结算可追踪
- [ ] 论场、议题、卡牌效果走统一配置
- [ ] 结算一致率100%
- [ ] 回放一致率100%

### 7.3 运营能力验收

- [ ] 论场分流有效（各论场>15%）
- [ ] 首赛季可稳定推出平衡补丁
- [ ] 异常对局可追溯
- [ ] 热修能力可用

## 八、结论

《百家争鸣》的核心竞争力在于**同步博弈机制**与**论场分流生态**。后期开发必须围绕一句话推进：

> **所有系统都要服务"明辩-暗谋-揭示-结算"的同步博弈体验，所有扩展都要强化"三席布阵+着书成长"的长期可玩性。**

如果某个功能不能强化这两条，就不进入首发关键路径。
