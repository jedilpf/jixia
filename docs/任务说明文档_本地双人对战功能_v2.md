# 《思筹之录》本地双人功能开发任务书 v2.0

## 一、任务背景

当前项目已经具备基础战斗界面、卡牌展示与部分战斗逻辑，但**缺少"本地双人对战"功能**。

### 现实问题
- 目前难以在同一台设备上快速验证双人对战流程
- 核心玩法（选门派、议题争夺、出牌、结算）无法被两名真实玩家完整体验
- 在正式联机之前，缺少一个低成本、可控、易调试的双人验证模式

### 任务目标
先补齐：**"同一设备 / 同一浏览器下的本地双人模式（Hot-seat / Pass-and-Play）"**

### 作用
1. 先把核心玩法跑通
2. 让两个人能在同一台电脑上真实打一局
3. 方便测试规则、UI、节奏、引导
4. 为后续联网模式提供可复用的战斗状态机和交互基础

## 二、具体要求

### 2.1 核心功能需求

#### 1）从主菜单进入"本地双人"
- 在主菜单中新增一个明确入口：
  - 开始对战
  - 本地双人
  - 规则说明
  - 开发测试入口

#### 2）本地双人局前流程
```
进入本地双人准备页
  ↓
展示本局议题预览（如果当前版本保留该步骤）
  ↓
玩家1选择门派
  ↓
切换遮罩页，提醒交给玩家2
  ↓
玩家2选择门派
  ↓
双方确认后进入战斗
```

#### 3）战斗中支持本地双人隐藏信息

**方式A：回合切换遮罩（优先推荐）**
- 每次轮到另一位玩家操作时，先出现一个全屏遮罩页
- 显示"当前请玩家1操作"或"当前请玩家2操作"
- 点击"准备好了"后进入该玩家的操作界面
- 手牌仅在当前玩家操作时显示

#### 4）支持一整局从开始到结束
- 双方选门派 → 进入战斗 → 抽牌 → 出牌 → 主议/旁议结算 → 中央议题推进/引爆判定 → 胜利判定 → 结果页展示 → 重新开始或返回主菜单

#### 5）所有状态变更必须真实反映在界面上
- 当前玩家是谁
- 当前回合数、当前费用、当前大势
- 当前议题状态
- 当前主议/旁议区内容
- 手牌数与手牌显示
- 战斗日志/状态提示

### 2.2 用户交互流程（建议MVP标准流程）

| Step | 阶段 | 说明 |
|------|------|------|
| 1 | 主菜单 | 玩家点击「本地双人」 |
| 2 | 议题预览 | 显示本局议题信息（若保留该模块） |
| 3 | 玩家1选门派 | 玩家1从候选门派中选择 |
| 4 | 交接遮罩页 | 显示"玩家1已完成选择，请将设备交给玩家2" |
| 5 | 玩家2选门派 | 玩家2从候选门派中选择 |
| 6 | 加载页 | 显示双方门派、本局议题、加载动画 |
| 7 | 战斗开始 | 进入战斗页，按当前战斗规则进行 |
| 8 | 回合切换遮罩 | 当需要切换当前操作玩家时，显示交接遮罩页 |
| 9 | 战斗结束 | 进入结果页，展示胜者、双方门派等 |

## 三、预期成果

### 3.1 功能成果
- 主菜单新增"本地双人"入口
- 本地双人局前流程完整可走通
- 本地双人战斗模式可完整打一局
- 存在"玩家交接遮罩页"用于保护隐藏信息
- 战斗结束后可正常进入结果页

### 3.2 组件/页面成果

| 类型 | 名称 | 说明 |
|------|------|------|
| 页面 | LocalBattleSetupScreen | 本地双人准备页 |
| 页面 | PlayerHandoverScreen | 玩家交接遮罩页 |
| 页面 | LocalBattleScreen | 本地双人战斗页 |
| 页面 | LocalBattleResultScreen | 本地双人结果页 |
| 组件 | TurnMaskOverlay | 回合遮罩覆盖组件 |
| 组件 | CurrentPlayerBanner | 当前玩家横幅 |
| 组件 | HiddenHandGuard | 手牌保护组件 |
| 组件 | LocalBattleActionBar | 本地双人操作栏 |

### 3.3 验收标准

**功能验收**
- 能从主菜单进入本地双人模式
- 两名玩家都能完成门派选择
- 可进入战斗并正常进行
- 每次轮换时手牌不会直接暴露给另一位玩家
- 能完成一局并进入结算页

**交互验收**
- 当前玩家身份明确
- 交接逻辑清楚，不混乱
- 按钮文案明确

**稳定性验收**
- 不出现回合错乱
- 不出现玩家1操作玩家2手牌的情况
- 不出现结算页无法返回的问题

## 四、技术约束

### 4.1 技术栈
- React
- TypeScript
- 组件化 UI
- 状态驱动渲染

### 4.2 架构原则

1. **规则逻辑与 UI 分离**
   - 战斗逻辑放在 `core/`
   - 数据配置放在 `data/`
   - 页面与组件只负责展示与触发 action
   - 不允许把规则写死在 React 组件里

2. **尽量复用现有战斗结构**
   - 本地双人模式不是做一套全新战斗系统
   - 而是在现有战斗骨架上补一个"同设备双人操作模式"

3. **不要大规模重构现有工程**
   - 如果能通过新增页面、补状态和增加切换遮罩完成，就不要整站重写

4. **先不实现联网**
   - 不要求：WebSocket、服务器房间、真实匹配、账号系统
   - 只做前端/本地状态驱动的一局双人对战

### 4.3 数据与状态要求

建议新增明确字段：
```typescript
type LocalBattleMeta = {
  mode: "local_pvp";
  currentActingPlayerId: string;
  pendingHandover: boolean;
  handVisibilityOwnerId: string | null;
};
```

## 六、建议实施步骤（分4个阶段）

### Phase 1：入口与基础状态
**目标**：主菜单接入"本地双人"，建立本地双人模式状态字段，新增基础页面骨架

**产出**：可从主菜单进入本地双人准备流程，页面切换骨架完整

### Phase 2：局前流程
**目标**：接入议题预览、玩家1/玩家2门派选择、交接遮罩页、加载页展示

**产出**：局前流程完整走通

### Phase 3：战斗内双人交接
**目标**：接入当前玩家标识、交接遮罩、当前玩家进入操作时才显示可见手牌

**产出**：一局内能正常轮换操作玩家

### Phase 4：完整对局闭环
**目标**：打完一局、胜利判定、结果页、返回主菜单/再来一局

**产出**：本地双人模式完整闭环可试玩

## 七、给 AI 助手的提示

### Prompt 约束模板

```
请严格只完成"本地双人模式"相关需求，不要额外扩展。
- 不要重构整个项目
- 不要修改未提及的核心规则
- 不要擅自新增联网逻辑
- 不要脑补未确认的玩法

请先列出你准备修改的文件清单，再开始改。

本次任务重点是：
1. 从主菜单进入本地双人
2. 两名玩家完成局前流程
3. 同一设备下通过"交接遮罩页"保护隐藏信息
4. 可以完整打一局并进入结算页

如果你发现文档中有模糊点，请先指出，不要自行决定。
```

### 建议分段提问方式（4个Prompt）

**Prompt 1**：只做主菜单入口 + 本地双人模式状态骨架

**Prompt 2**：只做局前流程（议题预览/双人选门派/交接遮罩）

**Prompt 3**：只做战斗内双人交接和手牌显示控制

**Prompt 4**：只做结果页和重新开始闭环

### 建议 AI 输出格式
要求 AI 输出时包含：
- 修改文件清单
- 每个文件的作用
- 改动摘要
- 还未实现的 TODO
- 是否存在风险或依赖项
