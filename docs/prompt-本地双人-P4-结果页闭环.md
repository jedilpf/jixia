# 本地双人功能开发 - Prompt 4：结果页与闭环

---

## Prompt 内容

```
## 任务目标

在《思筹之录》项目中实现"本地双人对战"功能的第四步（最后一步）：
**结果页 + 重新开始闭环 + 返回主菜单**

## 前置条件

Prompt 1、Prompt 2、Prompt 3 已完成：
- 主菜单入口已添加
- 局前流程已走通
- 战斗内交接和手牌控制已实现

## 具体要求

### 1. 胜利判定

确保在以下条件满足时正确判定胜负：
- 一方大势达到8点获胜
- 或满足其他胜利条件（如果有）

### 2. LocalBattleResultScreen 结果页

战斗结束后显示结果页，包含：

| 内容 | 说明 |
|------|------|
| 胜者信息 | "玩家X获胜！" |
| 双方门派 | 展示双方选择的门派 |
| 最终大势 | 各方大势数值 |
| 议题结果 | 最终议题状态（如果有） |
| 战斗时长 | 本局用时（可选） |

### 3. 重新开始闭环

在结果页提供：
- "再来一局"按钮 → 重新进入本地双人局前流程
- "返回主菜单"按钮 → 回到首页

### 4. 返回主菜单

确保可以正确返回主菜单：
- 清除本地双人相关状态
- 回到 HomeScreen

## 组件实现

### LocalBattleResultScreen

```typescript
interface LocalBattleResultScreenProps {
  winner: 'player1' | 'player2' | 'draw';
  player1Faction: string;
  player2Faction: string;
  player1Momentum: number;
  player2Momentum: number;
  onPlayAgain: () => void;
  onReturnHome: () => void;
}
```

### 样式要求

- 胜者高亮显示
- 双方信息对比清晰
- 按钮操作明确

## 约束

- 不要修改现有的 AI 对战逻辑
- 不要改变战斗规则
- 确保状态正确清理

## 输出要求

请先列出：
1. 准备修改的文件清单
2. 每个文件的作用
3. 改动摘要
4. 还未实现的 TODO（应该没有太多了）
5. 是否存在风险或依赖项

完成后，本地双人模式 MVP 应该完整可用！
```
