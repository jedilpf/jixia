# Codex 水印与任务留档规范 v1

> Author: Codex AI
> Watermark ID: CODEX-WM-20260328-V1
> Effective Date: 2026-03-28

## 1. 目标

避免“谁改的、改了什么、为什么改”在长期项目中丢失，确保多 AI 协作可追溯。

## 2. 何时必须执行

以下任务必须执行本规范：

1. 跨模块改动（例如入口、数据源、CI、音频链路、主流程收口）
2. 整项目完善类任务
3. 会影响其他 AI 执行口径的规则类任务

## 3. 开工前留档（强制）

开始实现前，必须先创建任务意图文档：

- 路径：`docs/project/record/task-intent-<date>-<task-id>.md`
- 模板：`docs/standards/task-intent-log-template.md`

必填项：

1. 本次目标
2. 允许修改文件范围
3. 禁止修改范围
4. 最小验证项
5. 退出条件（什么算完成）

## 4. 交付水印（强制）

所有正式交付文档必须包含以下最小水印头：

```md
> Author: Codex AI
> Watermark ID: CODEX-<task-id>-<yyyymmdd>
> Task ID: <task-id>
> Date: YYYY-MM-DD
```

说明：

1. 代码文件不强制加入注释水印，避免污染实现细节。
2. 代码交付的水印承载在“任务意图文档 + 提交说明/PR描述”中。

## 5. 验收清单

1. 是否存在任务意图文档
2. 是否存在水印头
3. 变更是否在声明范围内
4. 验证项是否执行并记录

## 6. 失败判定

出现以下任一情况即判定不合规：

1. 无任务意图文档即开始改动
2. 正式文档无水印头
3. 实际改动超出声明范围且未仲裁
