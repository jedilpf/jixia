# 争鸣叙史 V9 镜头语言与运镜约束文档 (Cinematography Convention)

## 核心设计哲学
在顶级的交互式叙事中，静态的背景图是破坏沉浸感的罪魁祸首。为了对标《底特律：变人》的电影感，V9 架构下的背景切换必须**告别单一的淡入淡出（Crossfade）**，引入**动态镜头语言（Cinematography）**。每一次场景切换，都必须附带特定的运镜轨迹，以暗示剧情的情绪走向。

---

## 一、 运镜词典 (Camera Motion Dictionary)
系统预设 6 种核心运镜模式，每一种模式对应特定的剧情情绪：

| 运镜指令 (Motion) | 镜头轨迹 (Transform) | 适用剧情场景 (Scenarios) | 情绪色彩 (Mood) |
| :--- | :--- | :--- | :--- |
| `tilt_up` | 从画面底部缓慢上移至顶部 (`y: [10%, -10%]`) | 探望病人（从床尾到床头）、仰望高耸建筑（问道堂）、审视 | 敬畏、压抑、绝望 |
| `dramatic_zoom` | 极速拉近然后缓慢放大 (`scale: [1, 1.2, 1.25]`) | 惊醒、遇袭、关键线索被发现、剑拔弩张 | 惊悚、紧张、顿悟 |
| `zoom_out` | 从局部特写缓慢拉远 (`scale: [1.3, 1]`) | 梦境醒来、视野开阔、从回忆拉回现实 | 释然、孤寂、全局观 |
| `pan_left` / `pan_right` | 水平方向缓慢平移 (`x: [5%, -5%]`) | 走廊漫步、街头视察、时间流逝 | 平缓、日常、探索 |
| `breathing_focus` | 原地极小幅度缩放 (`scale: [1, 1.05, 1]`) | 深刻的内心独白、面对面的对峙、凝视某物 | 极度专注、压迫感 |
| `slow_push` | 标准缓慢推进 (`scale: [1, 1.1]`) | 通用默认运镜、走向某人、进入新场景 | 沉浸、代入 |

---

## 二、 剧情-资产-运镜 映射规则 (Scene-to-Motion Mapping)
每一个 `contentBeat`（基于关键词触发的场景）都必须强制绑定一个 `CameraMotion`。

### 示例映射逻辑：
1.  **父亲临终场景** (`['父亲临终', '气息如游丝', '此信']`)
    *   **运镜**：`tilt_up` (模拟从床尾走到床头，或者目光从下往上绝望地看)。
2.  **恶梦惊醒** (`['冷汗', '心跳如鼓', '惊醒']`)
    *   **运镜**：`dramatic_zoom` (模拟心悸瞬间的瞳孔地震)。
3.  **初入稷下街道** (`['街道', '商贾', '人潮']`)
    *   **运镜**：`pan_right` (模拟走在街上左右环顾)。
4.  **云海扁舟/问道堂** (`['问道堂', '宫主', '梦中']`)
    *   **运镜**：`zoom_out` (模拟身处广阔云海的渺小感，或梦境的宏大)。

## 三、 技术实现约束 (Technical Implementation)

1.  **数据结构重构**：
    `getIllustration` 函数的返回值必须从单纯的 `string`（图片路径）升级为 `SceneAsset` 对象：
    ```typescript
    interface SceneAsset {
      asset: string;
      motion: CameraMotion;
    }
    ```
2.  **Framer Motion 动态注入**：
    禁止使用全局 CSS `animation` (如之前的 `kenburns`)。
    必须在 `<motion.img>` 中，基于传入的 `motion` 类型，动态计算 `animate` 和 `transition` 属性。
3.  **过渡时间 (Duration)**：
    *   常规运镜：`30s` ~ `40s`（极慢，不喧宾夺主）。
    *   戏剧性运镜（`dramatic_zoom`）：`0.5s` 极速推进，随后转为 `30s` 的缓慢变化。
4.  **画面冗余度**：
    为了支持平移和缩放而不露黑边，图片初始必须设定 `scale: 1.15` 或更大（具体由各运镜逻辑控制），确保 `objectFit: 'cover'` 始终充满屏幕。

---
**执行准则**：宁可夸张，不可平庸。每个画面的出现必须像电影导演亲自掌镜一样具有明确的目的性。
