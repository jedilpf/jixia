# 主界面按钮逆向拆解（先拆解，后重画）

更新时间：2026-04-03  
范围：`btn-start.png`、`btn-characters.png`、`btn-collection.png`

## 1) 先说结论
- 当前仓库没有 PSD/AI 等分层源文件，只有扁平化 PNG。
- 因此“图层数”只能做**结构化推断**，不能宣称绝对真实层数。
- 三个按钮属于同一美术体系，但存在两种版式模板：
  - 模板 A（紧凑横幅）：`btn-start`、`btn-collection`
  - 模板 B（长幅庆典）：`btn-characters`

## 2) 已核验基础数据
- `btn-start.png`：`795 x 288`
- `btn-collection.png`：`807 x 272`
- `btn-characters.png`：`1349 x 437`

## 3) 推断图层结构（按视觉职责，不是文件真实图层）

### 模板 A（开始辩斗 / 卡牌图鉴）推断为 8 层
1. 外发光与投影层（按钮整体悬浮感）
2. 外框金属牌匾层（柱、边、顶部徽饰）
3. 内面板底纹层（暗棋盘/纹理）
4. 主题色调层（整体暖金或青绿调色）
5. 左侧符号层（每个按钮独有主标识）
6. 标题文字层（主标题大字，发光描边）
7. 局部粒子/火花层（氛围细节）
8. 底部或边缘高光层（收边与体积）

### 模板 B（稷下人物志）推断为 9 层
1. 外发光与投影
2. 长幅主框架
3. 顶/底中心徽饰
4. 双侧垂挂装饰（灯笼/流苏）
5. 内面板底纹
6. 左侧独有符号层
7. 标题文字层
8. 粒子火花层
9. 边缘高光收边层

## 4) 共性设计规则（必须遵守）
- 规则 A：**一个按钮 = 一张完整图**。不使用运行时覆盖板遮字。
- 规则 B：标题字形必须同体系（古风厚重、发光描边、非现代 UI 字）。
- 规则 C：左侧符号必须唯一，不能与其他入口复用语义图形。
- 规则 D：统一“框体语言”，通过“符号+色调”区分功能，不通过结构突变区分。
- 规则 E：主标题为视觉中心，符号权重次之，装饰细节最后。

## 5) 针对“争鸣史”的重画约束（下一步执行）
- 使用模板 A（与开始辩斗/卡牌图鉴同级）。
- 左侧符号改为“史册/印玺语义”，禁止卷轴卡牌同构。
- 标题固定 `争鸣史`，字形与其他按钮保持同体系（发光+描边+厚重骨架）。
- 保留独立色相（青金冷调），但框体与体积关系不变。
- 输出仅 `public/assets/btn-story.png`，接入时仅替换图片引用，不加额外叠层。
