## 一、现有文档风格分析

### 1.1 标题格式规范

**现有文档标题格式**：
```
# 《谋天下·问道百家》xxx v1.2
# 知识库-05-xxx
```

**特点**：
- 主标题包含项目名称《谋天下·问道百家》
- 版本号直接标注在标题末尾
- 或使用"知识库-序号-主题"格式

### 1.2 文档头部格式

**现有格式**：
```markdown
## 文档说明

本文档整合了xxx，以v1.2版本为基准。

---

## 一、章节标题
```

**或**：
```markdown
**文档版本**: 2.0.0  
**更新日期**: 2026-03-09  
**适用范围**: 前端 / 服务端 / 策划配置 / QA / 数据

---
```

### 1.3 章节编号规范

| 层级 | 格式 | 示例 |
|------|------|------|
| 一级章节 | 中文数字 + 标题 | `## 一、核心规则` |
| 二级章节 | 阿拉伯数字 | `### 1.1 全局规则` |
| 三级章节 | 阿拉伯数字 | `#### 1.1.1 具体规则` |

### 1.4 表格使用规范

**现有文档表格格式**：
```markdown
| 规则项 | v1.0 | v1.2 |
|--------|-------|------|
| 大势获胜条件 | 6点 | **8点** |
```

**特点**：
- 表头使用加粗或普通文本
- 重要数据使用加粗强调
- 表格简洁，列数适中

### 1.5 代码块使用规范

**现有文档使用ASCII艺术图**：
```
┌─────────────────────────────────────────────────────────────┐
│                        回合开始                              │
├─────────────────────────────────────────────────────────────┤
│  1. 回合开始阶段                                             │
│     - 双方获得用度                                          │
└─────────────────────────────────────────────────────────────┘
```

### 1.6 专业术语使用

**游戏专用术语**：
- 辩锋、根基、用度、大势
- 主议、旁议、明辩、暗谋
- 立论、施策、反诘、门客、玄章
- 三席、文脉、着书

**技术术语**：
- TypeScript、React、Node.js
- camelCase、PascalCase、kebab-case

### 1.7 语言风格特点

| 特点 | 说明 |
|------|------|
| **简洁性** | 句子简短，信息密度高 |
| **专业性** | 使用专业术语，避免口语化 |
| **结构化** | 大量使用列表、表格、代码块 |
| **无emoji** | 现有文档不使用emoji符号 |
| **正式语气** | 陈述句为主，避免感叹号 |

## 三、调整建议

### 3.1 标题格式统一

**调整为**：
```markdown
# 《谋天下·问道百家》xxx v1.0

## 文档说明

本文档描述xxx，以v1.0版本为基准。

---

## 一、章节标题
```

### 3.2 移除emoji符号

**需要移除的emoji**：
- 📋、🎯、✅、❌、⚠️、🔴、🟡、🟢、⚪
- 🔍、📝、🔧、📚、🆘、📖、🎮、📊

**替换为文字**：
- ✅ → 完成/已实现
- ❌ → 未完成/问题
- ⚠️ → 注意/警告
- 🔴 → 紧急
- 🟡 → 重要
- 🟢 → 普通

### 3.3 表格格式统一

**调整为**：
```markdown
| 属性 | 值 |
|------|-----|
| 文档版本 | 1.0.0 |
| 更新日期 | 2026-03-23 |
| 适用范围 | 全项目文档 |
```

### 3.4 变更记录格式统一

**调整为**：
```markdown
## 变更记录

| 版本 | 日期 | 变更内容 | 作者 |
|------|------|----------|------|
| v1.0.0 | 2026-03-23 | 初始版本 | AI Assistant |
```

### 3.5 语言风格统一

- 移除口语化表达
- 使用正式陈述句
- 避免感叹号
- 保持简洁专业
