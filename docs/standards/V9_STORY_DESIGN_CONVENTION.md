# 争鸣叙史 V9 视觉与交互重构约束文档 (Cinematic Narrative Convention)

## 核心设计哲学
摒弃传统视觉小说（VN）“大黑框遮挡画面”的廉价感，汲取《底特律：变人》的沉浸式戏剧体验。UI 必须**去物质化、呼吸感、动态化**。每一次抉择都必须带有“改变历史”的命运重量感。

---

## 一、 空间图层架构 (Z-Index Hierarchy)
整个叙事界面必须严格划分为 6 个深度图层，形成裸眼 3D 般的视差效果：

### L0 - 寰宇层 (Atmosphere Layer | Z: 0)
*   **功能**：奠定场景基调的情绪背景。
*   **内容**：深渊黑玉色背景（`#0a0503`）、全局雾效（动态模糊）、随风场飘动的炽金粒子（金粉）。

### L1 - 视界层 (Cinematic Layer | Z: 10)
*   **功能**：场景插画展示。
*   **内容**：全屏或宽幅的场景 CG。必须具有极其缓慢的 `Ken Burns`（平移缩放）效果，禁止静态死图。

### L2 - 演像层 (Character Layer | Z: 20)
*   **功能**：名士（NPC）与主角的呈现。
*   **内容**：人物立绘。
*   **动效约束**：必须具备周期性的“呼吸动效”（极其微小的 Y 轴与 Scale 变化）。需带有基于 L1 环境色的动态光晕或边缘侧光。

### L3 - 叙事层 (Narrative Layer | Z: 30)
*   **功能**：承载文本信息。
*   **内容**：打破传统的“下半屏大黑框”。改用**悬浮式绢帛切片**或类似电影字幕的**极简沉浸式文本**。文本出现需带水墨晕染特效。

### L4 - 命轨层 (Destiny Layer | Z: 40) - 核心交互
*   **功能**：蝴蝶效应反馈与抉择。
*   **内容**：
    *   **抉择竹简**：选项不再是死板的按钮，而是以带透视感的“悬浮竹简”形式在屏幕空间展开。
    *   **回响通知（底特律特色）**：当好感度或名望变化时，在屏幕边缘出现类似底特律“软件不稳定性 ↑”的冷峻 UI 提示（例如：“儒家 羁绊加深 ✦”）。

### L5 - 统御层 (Control Layer | Z: 100)
*   **功能**：系统控制（不干扰叙事）。
*   **内容**：顶部系统栏（归园、存档、设置）。在无鼠标活动时应**自动隐退或极度透明化**，鼠标靠近时浮现。

---

## 二、 核心功能模块 (Modules)

### 1. 全息场景模块 (Holographic Scene Module)
*   **负责**：解析当前 `StoryNode` 的背景与人物数组。
*   **功能**：平滑地进行场景切换（水墨转场），管理立绘的进退场（禁止瞬间闪现，必须带有淡入/平移）。

### 2. 沉浸文本引擎 (Immersive Text Engine)
*   **负责**：解析 `scene`, `narration`, `inner_thought`, `dialogue`。
*   **功能**：
    *   **场景/旁白**：屏幕中央全屏排版，居中，带有深邃的黑玉渐变遮罩。
    *   **名士对话**：跟随立绘位置，采用“侧边悬浮对话框”或极简字幕，不再强制占用屏幕下半部分。
    *   **内心独白**：使用特殊的斜体、低饱和度色调，伴随心跳般的微光动效。

### 3. 蝴蝶抉择模块 (Butterfly Choice Module)
*   **负责**：渲染剧情分歧点（Choices）。
*   **功能**：
    *   选项出现时，L1-L3 图层变暗并带有极强的景深（Blur 效果），将玩家注意力强行拉入 L4。
    *   选项带有“炽金流光”悬停反馈。

### 4. 命轨回响模块 (Destiny Echo Module)
*   **负责**：替代原有的“底部静态好感度条”。
*   **功能**：
    *   取消常驻的底部能量条（太出戏，破坏电影感）。
    *   改为**事件触发式反馈**：仅在对应势力好感度发生变化时，在右上角或右下角以高级 UI 动画弹出（如：法家 信任度 +5）。
    *   在菜单中提供专门的“命轨界面”供玩家随时查看所有势力面板。

---

## 三、 设计约束与红线 (Design Redlines)
1.  **禁止纯色遮罩**：所有 UI 容器必须带有 `backdrop-blur`（毛玻璃）和透明度（最高不超过 0.85）。
2.  **绝对字体控制**：正文统一使用 `Noto Serif SC`。非必须情况，不使用无衬线体。
3.  **零静态规则**：屏幕上永远不能是完全静止的（必须有极微弱的背景粒子或呼吸动效维持生命感）。
