# 卡牌状态管理规范

## 1. 目标
- 防止因版本收口误删卡牌资产。
- 区分“当前版本可玩卡池”和“总卡池资产”。
- 让图鉴展示、运行时加载、策划台账使用同一套状态语义。

## 2. 核心原则
- 不删资产，只改状态：除重复文件/错误导入外，不因“暂时不用”删除卡牌。
- 图鉴不等于总卡池：默认图鉴只代表当前版本展示策略。
- 运行时按状态加载：当前版本默认仅加载 `active`。
- 状态优先于备注：`notes` 用于解释，不替代状态判定。

## 3. 状态枚举
- `active`：当前版本启用；可进入正式图鉴与运行时卡池。
- `planned`：已设计暂不上线；保留资产，不进当前对局。
- `draft`：草案中；规则未定稿，不进当前对局。
- `rework`：待重做；保留概念，不进当前对局。
- `archived`：历史归档；默认不展示，仅用于查档。

## 4. 展示与运行规则
- 图鉴默认筛选：仅显示 `active`。
- 图鉴扩展筛选：可选显示 `planned` / `draft` / `rework`，并标注“暂未开放/调整中”。
- `archived` 默认仅在开发者或策划档案视图可见。
- 运行时默认只读 `active`；非 `active` 不进入构筑、抽取、奖励池、AI 选牌流程。

## 5. 生命周期操作
- 新卡创建：默认 `draft` 或 `planned`，验证通过后可转 `active`。
- 临时下线：`active -> rework` 或 `active -> planned`，不删除文件。
- 明确停用但需保留历史：转 `archived`。

## 6. 删除边界
仅以下情况允许删除卡文件：
- 明确重复文件。
- 明确错误导入或测试垃圾数据。
- 已完成归档记录且确认不再保留该资产。

## 7. 当前执行基线（2026-03-24）
- `content/cards` 当前接入的 12 张卡设为 `active`，作为当前可玩基线。
- 后续非当前版本卡牌一律通过状态降级管理，不通过删除管理版本。
