## 为什么这个顺序？

### 为什么先做UI？

1. **用户第一眼看到的是UI** - 界面不清楚，用户会困惑
2. **现有功能已经能运行** - 但界面混乱影响体验
3. **不影响后续数值调整** - UI和数值是独立的
4. **相对独立** - 改UI不需要改核心逻辑

### 为什么数值在UI之后？

1. **需要稳定的测试环境** - UI稳定后才能准确测试数值
2. **数值调整需要多次试玩** - 试玩需要清晰的界面
3. **影响战斗体验** - 数值不合理会影响AI改进的评估

### 为什么后端延后？

1. **MVP不需要联网** - 单机版本足够演示核心玩法
2. **后端成本高** - 需要服务器、运维、安全
3. **本地双人可替代** - 同设备双人模式可实现"人人对战"
4. **可以迭代添加** - 后端不影响现有系统
