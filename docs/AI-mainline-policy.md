# AI 协作说明：当前主线基线（2026-04-01）

> 本文件已按 `CONFLICT-20260401-001` 仲裁结论更新。  
> 适用范围：当前本地分支与本地验收流程。若后续主线迁移，再创建新的冲突仲裁记录后更新本文件。

## 目标结论

1. **当前唯一默认主线**：`src/App.tsx` 本地主菜单链路（菜单 -> 过场 -> 战前 -> 战斗 + 剧情/人物志/图鉴）。
2. `MvpFlowShell`（`src/ui/screens/MvpFlowShell.tsx` + `src/app/*` + `src/core/*`）当前为并行保留链路，不作为默认入口。
3. 任何“主线切换”必须先完成冲突仲裁并满足迁移验收清单。

---

## 给其他 AI 的硬规则

1. 未经明确仲裁，不得将默认入口切到 `MvpFlowShell`。
2. 当前验收需求优先落在默认主线相关文件：
   - `src/App.tsx`
   - `src/components/MainMenu.tsx`
   - `src/components/BattleSetup.tsx`
   - `src/components/PreBattleFlow.tsx`
   - `src/components/BattleFrameV2.tsx`
   - `src/ui/screens/StoryScreen.tsx`
3. 涉及并行链路时，必须先写明“是否影响默认主线行为”。
4. 多 AI 并行场景下，禁止直接删除“看似多余”文件；先审查用途并记录结论。

---

## 入口策略（当前生效）

1. 默认打开项目：进入 `src/App.tsx` 主链路（无 URL 参数依赖）。
2. `MvpFlowShell` 仅作为并行开发链路，不用于默认验收路径。
3. 目标：
   - 先稳定用户当前可见链路。
   - 再进行可回滚的渐进式收敛/迁移。

---

## 主线迁移前置条件（必须全部满足）

1. 剧情、人物志、卡牌图鉴、战斗、返回导航在目标链路均可达且行为等价。
2. 至少完成一次完整人工链路验收（从主菜单到战斗结束再返回主菜单）。
3. 基础校验通过（至少 `npm run typecheck` 与指定任务门禁）。
4. 输出冲突仲裁记录，明确迁移收益、风险与回退方案。

---

## 推荐开发流程（多 AI 协作）

1. 先确认任务是否属于“当前默认主线”。
2. 改动前先列“拟修改文件清单”并标注是否触达并行链路。
3. 完成后执行最小门禁校验（任务级 `ai:finalize` + 必要运行验证）。
4. 在提交说明中明确：
   - 本次是否触达默认主线
   - 是否改变入口策略（正常应为“否”）

## 冲突仲裁机制（强制）

当出现以下情况时，任何 AI 都不得直接实现：

1. 两个 AI 给出互相冲突的方案。
2. 同一模块出现“接口设计 / 实现方式 / 数据结构”冲突。
3. 文档口径与代码现实冲突且会影响默认链路行为。

执行规则：

1. 先填写 `docs/standards/conflict-arbitration-template.md`。
2. 在记录中写明双方观点、仲裁依据、最终方案、实施要求。
3. 信息不全时，必须先向用户提问确认，再改动实现。
4. 仲裁结论未明确前，禁止进入实现阶段。
