# 出牌交互设计文档

## 一、核心交互原则

1. **所见即出**：玩家看到的战场区域 = 可以出牌的目标区域
2. **操作可逆**：出牌后可以取消，回到手牌
3. **直观反馈**：每一步操作都有即时视觉反馈

---

## 二、出牌操作流程

### 2.1 拖拽出牌（标准流程）

```
玩家从手牌拖拽卡牌
  → 拖拽过程中，目标议区高亮显示可放置区域
  → 释放卡牌到主议区或旁议区
  → 系统判断：
      费用足够？→ 出牌成功，卡牌从手牌移至议区
      费用不足？→ 出牌失败，卡牌弹回手牌
  → 显示结算信息（可用势/剩余势）
```

### 2.2 点击取消已出卡牌

```
玩家点击已放置在议区的卡牌
  → 该卡牌回到手牌
  → 恢复已消耗的势
  → 议区槽位空出
```

### 2.3 锁定机制（时间驱动）

```
倒计时归零时：
  → 自动锁定当前已放置的卡牌
  → 进入下一阶段（resolve 或下一 layer）
  → 锁定后不可取消
```

## 三、议区布局与可放置规则

### 3.1 议区结构

```
┌─────────────────────────────────────┐
│              敌方旁议                │
│          [槽位1][槽位2][槽位3]        │
├─────────────────────────────────────┤
│              敌方主议                │
│          [槽位1][槽位2][槽位3]        │
├─────────────────────────────────────┤
│          （议题/中央区域）            │
├─────────────────────────────────────┤
│              我方主议                │
│          [槽位1][槽位2][槽位3]        │
├─────────────────────────────────────┤
│              我方旁议                │
│          [槽位1][槽位2][槽位3]        │
└─────────────────────────────────────┘
```

### 3.2 可放置规则

| 卡牌类型 | 可放置议区 | 说明 |
|----------|-----------|------|
| 立论 | 主议 / 旁议 | 持续在场，形成场面 |
| 策术 | 主议 / 旁议 | 即时结算，不留场 |

**通用限制**：
- 每个议区最多 3 张卡（无论类型）
- 出牌需消耗势，不足则无法放置
- 同一回合同一张卡只能出现在一个议区

## 四、两层出牌阶段

### 4.1 Layer 1（第一手）

- **可操作时间**：40 秒（或 PHASE_DURATION.play_1）
- **可出牌数**：最多 1 张
- **可操作议区**：主议 / 旁议
- **锁定方式**：玩家主动点击"锁定"按钮，或倒计时归零

### 4.2 Layer 2（第二手）

- **可操作时间**：40 秒（或 PHASE_DURATION.play_2）
- **可出牌数**：最多 1 张（可与第一手不同）
- **可操作议区**：主议 / 旁议
- **锁定方式**：同上

### 4.3 阶段流转

```
Layer 1 出牌 → Layer 1 结算（resolve_1）
  → Layer 2 出牌 → Layer 2 结算（resolve_2）
    → 回合结束 → 议区结算（辩锋对比）
      → 进入下一回合
```

---

## 五、势（费用）消耗规则

### 5.1 基础消耗

```
出牌消耗势 = 卡牌费用（cost）
```

### 5.2 筹（Chou）的使用

- 旁议胜者获得 1 个筹（上限 1）
- 出牌时自动使用筹（如果可用）：实际消耗 = 费用 - 1
- 筹不可主动选择使用/不使用，自动生效

### 5.3 取消出牌时的势恢复

```
取消已出卡牌 → 返还所消耗的势
```

---

## 六、策术（即时效果）处理

### 6.1 出牌时即触发

策术牌放置到议区时，**立即触发效果**，不等待结算阶段。

### 6.2 策术目标

| 策术效果 | 默认目标 |
|----------|---------|
| 伤害 | 对手所在议区（与放置位置对应） |
| 治疗/护盾 | 自身所在议区 |
| 抽牌 | 自身手牌 |
| 弃对方牌 | 对手手牌 |

### 6.3 策术效果示例

| 策术名 | 效果 | 目标 |
|--------|------|------|
| 驳杂去芜 | 造成 3 点伤害 | 对手所在议区 |
| 收束成文 | 抽 2 张牌 | 自身手牌 |
| 旁征博引 | 获得 3 护体 | 自身 |

## 七、视觉与交互反馈

### 7.1 可拖拽状态

- 悬停手牌 → 卡牌轻微放大（scale 1.05）
- 拖拽中 → 卡牌跟随指针，透明度 0.8
- 可放置区域 → 高亮边框（绿色/金色）
- 不可放置区域 → 红色遮罩

### 7.2 出牌成功

- 卡牌动画移入议区槽位
- 消耗势的数值飘字（-X）
- 议区槽位边框闪烁确认

### 7.3 出牌失败

- 卡牌弹回手牌位置（带动画）
- 势不足提示（"势不足"飘字）

### 7.4 点击取消

- 点击已放置卡牌
- 确认提示（"取消此牌？"）
- 确认后卡牌返回手牌，势返还

### 7.5 锁定后状态

- 已锁定卡牌不可点击
- 锁定动画（边框锁定图标）
- 倒计时结束自动进入下一阶段

## 九、技术实现要点

### 9.1 拖拽实现

推荐使用 HTML5 Drag & Drop API 或 react-dnd 库。

关键事件：
- `onDragStart`：记录拖拽的卡牌 ID
- `onDragOver`：判断目标槽位是否可放置
- `onDrop`：执行出牌逻辑，消耗势
- `onDragEnd`：拖拽取消时清理状态

### 9.2 状态管理

```
BattlePlayer.plan = {
  layer1CardId: string | null,    // 第一手卡牌
  layer1TargetSeat: 'zhu_yi' | 'pang_yi',
  layer2CardId: string | null,    // 第二手卡牌
  layer2TargetSeat: 'zhu_yi' | 'pang_yi',
  lockedLayer1: boolean,
  lockedLayer2: boolean,
  usedCost: number,               // 已消耗势
}
```

### 9.3 取消操作映射

| 操作 | 触发的 Engine Action |
|------|---------------------|
| 拖拽出牌（Layer1） | `PLAN_CARD { slot: 'layer1', cardId }` |
| 拖拽出牌（Layer2） | `PLAN_CARD { slot: 'layer2', cardId }` |
| 取消已出卡牌（Layer1） | `PLAN_CARD { slot: 'layer1', cardId: null }` |
| 取消已出卡牌（Layer2） | `PLAN_CARD { slot: 'layer2', cardId: null }` |
| 锁定 Layer1 | `LOCK_LAYER1` |
| 锁定 Layer2 | `LOCK_LAYER2` |

### 9.4 费用验证伪代码

```typescript
function canPlayCard(player: BattlePlayer, card: DebateCard): boolean {
  const rawCost = card.cost;
  const availableCost = player.resources.cost;

  // 基础费用检查
  if (rawCost > availableCost) return false;

  // 筹折扣检查
  const effectiveCost = (player.resources.chou > 0) ? rawCost - 1 : rawCost;
  if (effectiveCost > availableCost) return false;

  return true;
}
```
