# 元宇宙方向发展总方案 v1

更新时间：2026-04-02  
负责人：Codex（大框架与复杂系统）  
协作分工：Antigravity（美术体系）、Trae（文案体系）

---

## 1. 目标定义与边界

### 1.1 最终方向（北极星）
我们把“元宇宙”定义为：
- 同一身份长期存在（跨设备、跨赛季、跨玩法）
- 同一世界持续演进（世界事件可沉淀）
- 同一资产可成长（卡牌/角色/称号等有历史）
- 同一社交关系可迁移（好友、战队、师承、论坛声望）

### 1.2 执行总原则（保留当前内容）
- 保留现有战斗内核：同步回合、三席双层、五类卡结构不推倒。
- 保留现有内容基线：当前卡池与剧情链路不重做，只增量扩展。
- 保留现有代码入口：前端继续以 `src/` 为主，后端继续在 `server/` 迭代。
- 新能力采用外挂式架构：通过新服务模块接入，不动旧核心逻辑。

### 1.3 不做事项（当前版本）
- 不直接做重 VR/3D 客户端改造。
- 不在当前阶段引入高风险自由交易。
- 不做一次性“全量重构”。

---

## 2. 当前能力盘点（可直接复用）

### 2.1 可复用资产
- 战斗系统：已有可运行链路与规则。
- 剧情系统：已有节点、存档、章节进度结构。
- 后端基础：已具备 API 框架、进度与存档持久化能力。
- 策划与知识库：文档沉淀完整，便于执行一致性。

### 2.2 必补短板
- 统一身份层（账号、设备绑定、会话管理）。
- 社交与世界层（大厅、房间、世界事件、公告系统）。
- 经济与治理层（资产账本、赛季结算、风控审计）。
- 线上运营层（活动编排、埋点、监控、异常告警）。

## 4. 18个月路线图（按阶段推进）

## 阶段A：在线底座（2026-04-02 ~ 2026-06-15）
目标：从“本地可玩”升级到“可持续在线”。

### 交付
- 统一玩家身份（游客ID -> 正式账号绑定）。
- 云存档、云进度、跨设备读档。
- 关键埋点（留存、回流、战斗完成率、剧情完成率）。
- 运维基础：日志、错误追踪、告警阈值。

### 分工
- Codex：身份服务、存档一致性、API 契约、灰度发布策略。
- Antigravity：账号与个人页视觉稿、状态反馈组件（成功/失败/同步中）。
- Trae：账号引导文案、权限提示、存档冲突提示文案。

### 验收指标
- 跨设备读档成功率 >= 98%。
- 存档冲突可恢复率 >= 99%。
- 首次登录流程完成率 >= 90%。

---

## 阶段B：共享世界 MVP（2026-06-16 ~ 2026-10-15）
目标：让玩家开始“共同处于一个世界”。

### 交付
- 学宫大厅（在线列表、公告、世界事件入口）。
- 房间系统（创建/加入/观战/回放链接）。
- 世界频道（受限发言+反垃圾规则）。
- 每周世界事件（固定模板，先从轻量开始）。

### 分工
- Codex：大厅与房间后端、事件调度器、并发稳定性。
- Antigravity：大厅与房间 UI 体系、世界事件视觉素材模板。
- Trae：世界公告、事件叙事、频道引导文案与社区公约。

### 验收指标
- 大厅在线稳定时长 >= 99.5%。
- 房间创建成功率 >= 99%。
- 每周活动参与率（活跃用户）>= 35%。

---

## 阶段C：资产与赛季循环（2026-10-16 ~ 2027-03-31）
目标：形成长期成长驱动与内容循环。

### 交付
- 资产ID体系（卡牌成长、角色称号、论场成就）。
- 赛季机制（赛季任务、赛季奖励、重置与继承规则）。
- 受控交易试点（白名单、限频、审计日志）。
- 风控规则（异常行为检测、回滚与封禁流程）。

### 分工
- Codex：资产账本、交易风控、赛季结算与反作弊基础。
- Antigravity：赛季视觉主题、奖励展示动画规范、资产展示页。
- Trae：赛季叙事包装、任务文案、奖励命名与稀有度语言体系。

### 验收指标
- 赛季次月留存提升 >= 8%。
- 交易异常拦截准确率 >= 95%。
- 资产数据一致性事故 = 0（P0）。

---

## 阶段D：UGC 与治理体系（2027-04-01 ~ 2027-09-30）
目标：从“官方内容驱动”升级到“官方+玩家共创”。

### 交付
- 赛事工具（赛程、报名、回放、战报）。
- 轻量UGC（自定义房规模板、活动模板）。
- 审核系统（内容审核、举报、申诉、仲裁记录）。
- 创作者激励（贡献积分、荣誉体系、联动资源位）。

### 分工
- Codex：UGC 权限系统、审核后端、治理策略引擎。
- Antigravity：UGC 编辑器视觉框架、创作者中心 UI。
- Trae：创作规范、治理规则文本、社区运营话术体系。

### 验收指标
- UGC 月活创作者占比 >= 8%。
- 举报处理时效（24小时内）>= 95%。
- 社区负面事件收敛周期 <= 72小时。

## 6. 技术大框架（由 Codex 主导）

### 6.1 架构策略：核心不动，外层扩展
- Core Layer（保留）：战斗引擎、剧情引擎、基础卡牌数据。
- Meta Layer（新增）：身份、社交、世界事件、资产账本、治理。
- Ops Layer（新增）：日志、监控、告警、审计、灰度发布。

### 6.2 模块拆分建议
- `server/modules/identity`
- `server/modules/cloud-save`
- `server/modules/social-lobby`
- `server/modules/world-event`
- `server/modules/economy-ledger`
- `server/modules/governance`

### 6.3 兼容策略
- 所有新功能使用 feature flag，默认可关闭。
- 所有旧接口保留兼容期，新增接口走 `/api/v2/*`。
- 所有高风险模块必须先 shadow run，再切主链路。

---

## 7. 内容体系策略（由 Trae + Antigravity 联合落地）

### 7.1 文案体系（Trae）
- 建立统一术语表：系统文案、剧情文案、活动文案分层管理。
- 建立“世界事件模板库”：每周可复用，减少重写成本。
- 建立“品牌语气规范”：公告/提示/奖励文案统一风格。

### 7.2 美术体系（Antigravity）
- 建立 UI 设计令牌（颜色、间距、层级、状态）。
- 建立角色与学宫场景母版，减少重复产出。
- 建立活动视觉模板（banner、卡面、奖励展示）。

### 7.3 联动机制（Trae + Antigravity + Codex）
- 每周一次“内容-技术对齐会”：只确认可上线内容，不做泛讨论。
- 文案与视觉必须先过模板校验，再进入开发接入。
- 上线包统一包含：文本包、视觉包、配置包、回滚包。

## 8. 风险与止损机制

## 8.1 高风险清单
- 身份与资产一致性事故（P0）。
- 新系统拖慢战斗核心体验（P0）。
- 内容生产速度低于上线节奏（P1）。
- 社区失控（辱骂/外挂/刷子）导致口碑下滑（P1）。

## 8.2 止损策略
- 任何 P0 事故触发“功能降级开关”，优先保住可玩主链路。
- 赛季与交易模块发生异常时，自动切换只读模式。
- 世界事件可暂停，不影响战斗与剧情主循环。
- 每次大版本必须有“1键回滚”预案和演练记录。

---

## 9. 版本治理与协作规则

- 主分支只保留一套稳定可运行版本。
- 探索性功能在独立分支迭代，通过后再合并。
- 文档、接口、实现必须三者一致，不一致时以接口契约为准。
- 每个阶段结束必须形成“阶段复盘文档+下阶段准入条件”。

## 11. 本方案成功标准

- 不是“做了很多功能”，而是“旧内容不崩 + 新世界能持续跑 + 团队分工长期稳定”。
- 每个阶段都可独立上线、独立运营、独立回滚。
- 到阶段D结束时，项目从“单机化体验”完成到“持续在线世界”转型。
