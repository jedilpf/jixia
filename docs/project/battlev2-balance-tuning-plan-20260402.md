# battleV2 数值优化策划案（2026-04-02）

## 1. 目标与边界

### 目标
- 降低当前 AI 的压制感，向「玩家胜率 45%-55%」区间收敛。
- 保证任意对局都可收束，不再出现无限拖局风险。

### 边界
- 不改卡池来源链路（保持 `cardsSource`，保持 170 张活跃卡参与演算）。
- 不做架构重构，只改 battleV2 引擎里的数值与终局兜底逻辑。

## 3. 回合上限裁决规则（新增）

当达到 `maxRounds` 且未自然结束时，按以下顺序裁决胜负：
1. 大势（`daShi`）高者胜。
2. 若大势相同，心证（`xinZheng`）高者胜。
3. 若仍相同，三席位总血量高者胜。
4. 以上都相同则平局。

这套规则的目的：既保证可收束，又尽量保留“局内经营成果”的公平表达。

## 4. 验收口径（本轮）

- P0（收束）：
  - 有固定回合上限并有明确终局裁决文案。
  - 终局会产生日志，便于复盘。
- P1（平衡）：
  - 先完成“压制度降档”的参数调整，不直接大改 AI 算法。
  - 下一轮通过扩样（建议 10 局起）观察胜率回归到 45%-55%。

## 5. 风险与后续

- 当前参数属于第一轮降压，不保证一次命中目标胜率区间。
- 建议后续用固定随机种子做分批扩样（10/30/100 局分层），每轮只调 2-3 个参数，避免多变量互相掩盖。
