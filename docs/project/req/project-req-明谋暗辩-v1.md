## 一、任务目标

### 1.1 核心目标

实现《百家争鸣》的"明谋暗辩"同步博弈机制，让游戏从"轮流出牌"转变为"同步提交、隐藏信息、固定结算"的策略博弈模式。

### 1.2 一句话定义

> **明谋暗辩 = 双方同时决定"把哪张牌放到哪个槽位"，但在揭示前，只能看到对方"放了牌、放在哪个槽位"，看不到具体是什么牌。**

### 1.3 要解决的核心问题

| 问题 | 传统模式 | 明谋暗辩模式 |
|------|----------|-------------|
| 信息博弈 | 看到对方出牌再反应 | 根据公开信息预判对方意图 |
| 策略深度 | 线性决策 | 设局、猜位、心理战 |
| 游戏节奏 | 轮流等待 | 同步操作、紧凑结算 |

## 三、核心功能需求

### 3.1 回合阶段流转

```
回合开始 → 同步提交(20s) → 锁定 → 揭示 → 固定结算 → 回合结束
```

| 阶段 | 时长 | 核心操作 | 信息可见性 |
|------|------|----------|-----------|
| 回合开始 | - | 抽牌、资源恢复 | 完全公开 |
| 同步提交 | 20秒 | 提交动作到槽位 | 仅见对方zone |
| 锁定 | - | 禁止修改 | 广播简报 |
| 揭示 | - | 公开具体卡牌 | 完全公开 |
| 固定结算 | - | 六层顺序结算 | 完全公开 |

### 3.2 同步提交机制

#### 3.2.1 动作类型

```typescript
type PlannedAction =
  | { type: 'play_card'; cardId: string; zone: 'main' | 'side' | 'prep'; useToken: boolean }
  | { type: 'pass' };
```

#### 3.2.2 提交槽位

| 槽位 | 允许牌类 | 作用 |
|------|----------|------|
| main（主议） | 立论、施策 | 主议争夺 |
| side（旁议） | 立论、施策 | 旁议争夺 |
| prep（预备区） | 施策 | 资源/全局效果 |

#### 3.2.3 合法性校验

- [ ] 卡牌是否在手牌中
- [ ] zone是否合法
- [ ] 牌类与zone是否匹配
- [ ] 当前zone是否允许放置
- [ ] 费用是否足够

### 3.3 信息隐藏规则

#### 3.3.1 提交阶段可见信息

| 信息 | 我方可见 | 对方可见 |
|------|----------|----------|
| 是否已提交 | ✅ | ✅ |
| 提交到哪个zone | ✅ | ✅ |
| 具体卡牌ID | ✅ | ❌ |
| 卡牌数值/效果 | ✅ | ❌ |
| 是否使用筹 | ✅ | ❌ |

#### 3.3.2 揭示后可见信息

| 信息 | 我方可见 | 对方可见 |
|------|----------|----------|
| 具体卡牌ID | ✅ | ✅ |
| 卡牌数值/效果 | ✅ | ✅ |
| 结算过程 | ✅ | ✅ |

### 3.4 固定顺序结算

```
Layer 1: 立论进场
    ↓
Layer 2: 施策生效
    ↓
Layer 3: 议位胜负结算
    ↓
Layer 4: 根基变化/离场
    ↓
Layer 5: 大势/筹更新
    ↓
Layer 6: 胜利检查
```

### 3.5 议位结算规则

#### 3.5.1 主议结算

- 比较双方主议立论最终辩锋
- 高者赢主议，+1大势
- 输者主议立论-1根基

#### 3.5.2 旁议结算

- 比较双方旁议立论最终辩锋
- 高者赢旁议，获得1筹
- 每人最多持有1筹
- 输者旁议立论-1根基

#### 3.5.3 平局处理

- 双方辩锋相等时无人获胜
- 双方该议位立论各-1根基

### 3.6 胜利条件

| 条件 | 说明 |
|------|------|
| 大势胜利 | 先到8大势获胜 |
| 心证归零 | 备选条件，心证归零即败 |

## 五、数据模型

### 5.1 核心类型定义

```typescript
type DebatePhase = 'turn_start' | 'submit' | 'locked' | 'reveal' | 'resolve' | 'turn_end' | 'finished';

type Zone = 'main' | 'side' | 'prep';

interface PlannedAction {
  type: 'play_card' | 'pass';
  cardId?: string;
  zone?: Zone;
  useToken: boolean;
}

interface PublicActionInfo {
  submitted: boolean;
  zone: Zone | null;
}

interface RevealInfo {
  player: { cardId: string; zone: Zone } | null;
  enemy: { cardId: string; zone: Zone } | null;
}

interface Resources {
  xinZheng: number;    // 心证
  lingShi: number;     // 灵势（费用）
  daShi: number;       // 大势
  chou: number;        // 筹
}
```

interface BattleState {

}

### 6.1 功能验收

- [ ] 同步提交阶段双方可同时操作
