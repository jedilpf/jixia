# 三档星级系统实现说明（2026-04-02）

## 本轮已生效

1. 卡牌元数据已接入 battleV2 运行时对象：
- `rarity`
- `starTier`（1/2/3）
- `unlockLevel`

2. 已接入等级解锁过滤：
- 战斗建立牌库时会根据 `playerLevel` 过滤未解锁星级卡。
- 当前 BattleFrameV2 会把玩家等级传入 battle 引擎（敌我同等级）。

3. 图鉴已接入星级与解锁提示：
- 显示星级徽记（★/★★/★★★）。
- 显示“Lv.X 解锁”文案。
- 顶部展示“当前等级 + 已解锁数量”。

4. 星级系统工具函数已落地：
- 稀有度到星级映射。
- 星级解锁等级判定。
- 牌组配额校验（2★/3★）工具函数。

## 本轮新增：硬性约束已接入

1. 牌库生成阶段已强制应用 `2★/3★` 配额：
- 按等级段执行配额（Lv1-14: 2★=0,3★=0；Lv15-29: 2★<=8；Lv30+: 2★<=8,3★<=3）。
- 配额超限的卡在建牌时被确定性裁剪，不会进入最终牌库。

2. 对局启动前新增配额拦截：
- 引擎会对我方与敌方牌库执行 `validateDeckTierQuota`。
- 任一方不合规时，直接阻止对局开始并写入明确日志（配额拦截原因）。

3. 图鉴顶部新增当前等级配额提示：
- 显示“2★ 上限、3★ 上限”，便于玩家理解当前可用边界。

## 下一步建议

1. 在构筑入口接入“逐张选卡实时计数”，让超限在拖牌阶段就被阻止。
2. 增加“超限替换推荐”（自动建议替换为同费用1★/2★牌）。
3. 把升级结算与等级持久化（本地/服务端）闭环，保证长期成长可见可用。
