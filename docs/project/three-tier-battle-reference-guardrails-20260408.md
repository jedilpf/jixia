- 回应“参考《月圆之夜》公开玩法信息，但不能照搬版权和结构”的要求。
- 统一后续三阶卡牌设计边界，避免把 `battleV2` 演化成外部同构系统。

## 3. 可借鉴的高层原则
### 3.1 强内容要渐进开放
- 高等级卡牌不应在早期一次性全部进入环境。
- 玩家应能清楚理解“为什么现在能用 / 还不能用”。

### 3.2 强卡要有清晰触发窗口
- 强效果应围绕有限时机、有限条件、有限数量展开。
- 高阶卡的强度应该来自“关键时点改变局势”，而不是全局无差别堆值。

### 3.3 成长应绑定节奏与风险
- 三阶卡的收益应和论场阶段、议题、心证/大势推进、明谋暗辩时机挂钩。
- 避免把成长理解成单纯的战前数值膨胀。

## 4. 明确拒绝的外部结构
- 商店刷新、商店等级、购买经验等 tavern/shop 循环
- 三连合成、金卡升级、joker 替代等合成升级循环
- 六格自动站位、固定攻击序列、镜像互殴式 auto-battler 结构
- 直接复用外部的卡牌分类、资源命名、回合骨架和文案包装

## 5. 我们自己的三阶结构锚点
- 三阶仍以仓库既有规则为主：
  - `starTier`
  - `unlockLevel`
  - `playerLevel`
  - `deck tier quota`
- 战斗骨架仍以 `battleV2` 既有阶段为主：
  - `ming_bian -> an_mou -> reveal -> resolve`
- 如果后续强化三阶“运作感”，只能挂接在我们自己的系统变量上：
  - `topic`
  - `writing`
  - 阶段时机
  - 阵营/技能文本

## 6. 对后续设计的直接约束
- 1 阶卡负责稳定底盘、基础连段和教学识别。
- 2 阶卡负责节奏变化、连动深化和中期转折。
- 3 阶卡负责低数量、高识别度、强时机性的“局势改写”。
- 任何新提案如果出现“战斗内升级商店”“卡牌合成升星”“外部同构资源循环”，都必须重新发起冲突仲裁。

## 7. 建议的后续推进方式
- 先在现有三阶体系内补“触发窗口”和“阶段身份”，而不是补新外循环。
- 先做卡牌分层职责与技能触发边界，再决定是否需要更深的成长反馈。
- 先用数值模拟验证三阶出场节奏，再讨论更大机制扩展。
