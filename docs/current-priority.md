## 🔴 P1：改进AI对手智能度（阻塞可玩性）

**为什么优先**：当前AI规划过于简单，无战术意识，严重影响游戏体验。这是"能玩"和"好玩"的分界线。

**目标**：
- AI在明辩阶段能考虑"对手可能出什么"
- AI在选秘策时有防守意识（预见对方高伤害卡）
- AI不会做明显的亏损选择（用高费卡打低威胁目标）

**工作量**：12-16小时  
**预计完成**：本周内  

**相关文件**：
- `src/battleV2/engine.ts` - callAI_AUTO_PLAN 函数
- `src/battleV2/laneSystem.ts` - 控制权评估

**验收标准**：
- 对同样的阵容，AI作出的决策不再是"任意选择"
- 肉眼看起来AI考虑了对手的威胁
- 玩一局20回合，AI至少赢一次（目前胜率接近0）

## 与数据契约的关系

全部代码改动涉及的数据结构，都必须遵守 `docs/data-contract.json` 的定义。
- Resources 的字段名 (xinZheng, lingShi, 等) — 与代码完全一致
- DebateCard 的 effectKind 枚举值 — 与engine.ts的定义同步
- DebateBattleState 的phase值 — 与代码的phase enum同步

这样做的好处：
- 如果某个字段改名，立刻在contract中发现冲突
- AI任务和代码审查都有"真相源"参考
- 新开发者上手时有清晰的数据定义
