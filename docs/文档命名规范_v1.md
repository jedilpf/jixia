# 《谋天下·问道百家》文档命名规范 v1.0

## 一、命名格式标准

### 1.1 标准格式

```
[模块]-[类型]-[主题关键词]-[版本].扩展名
```

### 1.2 格式说明

| 组成部分 | 说明 | 示例 |
|----------|------|------|
| 模块 | 功能模块标识 | battle、card、ui、system |
| 类型 | 文档类型标识 | design、spec、guide、report |
| 主题关键词 | 文档核心内容 | 核心规则、卡牌设计、界面框架 |
| 版本 | 版本号（可选） | v1.0、v1.2 |

### 1.3 命名原则

1. **使用中文**：主题关键词使用中文，便于理解
2. **简短精炼**：总长度不超过50字符
3. **无空格**：使用连字符 `-` 分隔各部分
4. **统一编码**：全部使用 UTF-8 编码

| 模块标识 | 中文名 | 说明 |
|----------|--------|------|
| battle | 战斗系统 | 战斗机制、结算、同步对战 |

| dev | 开发文档 | 技术架构、开发者指南 |
| meta | 元文档 | 索引、说明、维护规范 |

## 三、类型标识定义

| 类型标识 | 中文名 | 说明 |
|----------|--------|------|
| design | 设计文档 | 游戏设计、机制设计 |
| spec | 规格文档 | 技术规格、功能规格 |
| guide | 指南手册 | 开发者指南、用户指南 |
| report | 报告文档 | 测试报告、分析报告 |
| record | 记录文档 | 开发记录、优化记录 |
| ref | 参考资料 | 历史版本、原始资料 |
| draft | 草稿文档 | 早期版本、待定内容 |

## 四、版本号规范

### 4.1 版本格式

```
v主版本号.次版本号
v1.0、v1.2、v2.0
```

### 4.2 版本语义

| 版本类型 | 说明 |
|----------|------|
| v1.0 | 初始版本 |
| v1.x | 小幅更新、功能增强 |
| v2.0 | 重大版本迭代 |

### 4.3 版本使用规则

- 正式文档必须包含版本号
- 草稿和参考资料可不包含版本号
- 归档文档在文件名中保留原始版本号

## 五、命名示例

### 5.1 正确示例

```
battle-design-核心规则-v1.2.md
battle-spec-同步对战机制.md
card-design-门派设定全集-v1.2.md
ui-design-界面框架-v1.md
dev-guide-开发者指南.md
project-report-需求分析-v1.md
```

### 5.2 错误示例

```
# 包含空格
battle design 核心规则.md

# 过长
battle-design-这是非常长的描述性文字用来解释这个文档的具体内容.md

# 缺少模块标识
核心战斗规则.md
```

## 六、目录结构规范

### 6.1 目录命名

```
docs/
├── meta/               # 元文档
├── battle/             # 战斗系统文档
├── card/               # 卡牌系统文档
├── faction/            # 学派系统文档
├── ui/                 # 用户界面文档
├── system/             # 通用系统文档
├── project/            # 项目管理文档
├── dev/                # 开发技术文档
├── archive/            # 归档文档
│   ├── ref/            # 参考资料
│   └── old/            # 历史版本
└── external/           # 外部参考资料
```

### 6.2 文档放置规则

| 文档类型 | 放置目录 |
|----------|----------|
| 索引、规范等元文档 | docs/meta/ |
| 战斗相关设计文档 | docs/battle/ |
| 卡牌相关设计文档 | docs/card/ |
| 学派/门派设定文档 | docs/faction/ |
| 界面设计文档 | docs/ui/ |
| 开发指南、架构文档 | docs/dev/ |
| 项目管理文档 | docs/project/ |
| 测试报告、分析报告 | docs/project/report/ |
| 归档的历史文档 | docs/archive/ref/ 或 docs/archive/old/ |
