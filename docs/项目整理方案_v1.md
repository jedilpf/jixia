## 三、项目内部存在的问题

### 3.1 架构层面

**问题1：双套战斗系统并存**
- `src/battleV2/` - 同步对战V2系统
- `src/core/` - 旧版游戏引擎
- `src/components/BattleLayout.tsx` - 旧版布局
- `src/components/BattleFrameV2.tsx` - 新版布局

**影响**：维护成本高，新旧功能可能冲突

**问题2：MVP新流程与旧流程并行**
- `src/ui/` - 新MVP流程
- `src/components/BattleLayout.tsx` - 旧流程入口
- `App.tsx` 通过URL参数切换

**问题3：嵌套项目副本**
- 存在 `jixia2.0【完整链路版本】/jixia2.0【完整链路版本】/` 嵌套目录
- 包含部分源码副本

### 3.2 文件组织层面

**问题4：组件位置混乱**
```
src/components/
├── BattleFrameV2.tsx          # 根目录
├── BattleLayout.tsx            # 根目录
├── CardShowcase.tsx            # 根目录
```

**问题5：数据类型分散**
- `src/data/cards.ts`
- `src/data/showcaseCards.ts`
- `src/data/game/cards.ts`
- `src/battleV2/cards.ts`

- 与游戏项目代码无关
