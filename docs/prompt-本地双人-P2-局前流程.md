# 本地双人功能开发 - Prompt 2：局前流程

---

## Prompt 内容

```
## 任务目标

在《思筹之录》项目中实现"本地双人对战"功能的第二步：
**局前流程（议题预览/双人选门派/交接遮罩/加载页）**

## 前置条件

Prompt 1 已完成：
- 主菜单"本地双人"入口已添加
- LocalBattleSetupScreen 等页面骨架已创建
- 本地双人状态字段已建立

## 具体要求

### 1. 议题预览

在 LocalBattleSetupScreen 中添加议题预览步骤（如果当前版本有此模块）：
- 显示本局议题信息
- 玩家可以了解本局议题方向

### 2. 玩家1门派选择

玩家1从候选门派中选择：
- 复用现有的门派选择组件
- 选择后进入交接遮罩

### 3. 交接遮罩页（PlayerHandoverScreen）

当玩家1完成选择后，显示交接遮罩：
- 内容："玩家1已完成选择，请将设备交给玩家2"
- 按钮："继续"
- 点击后进入玩家2选择页

### 4. 玩家2门派选择

玩家2从候选门派中选择：
- 与玩家1相同的选门派逻辑
- 选完后进入加载页

### 5. 加载页

显示：
- 双方门派名称
- 本局议题
- 加载动画
- 点击后进入战斗

## 组件实现

### TurnMaskOverlay 组件

```typescript
interface TurnMaskOverlayProps {
  visible: boolean;
  currentPlayer: 1 | 2;
  message?: string;
  onReady: () => void;
}
```

功能：
- 全屏遮罩
- 显示当前请哪位玩家操作
- "准备好了"按钮
- 点击后进入操作界面

## 约束

- 不要重构整个项目
- 复用现有的 PreBattleFlow 门派选择逻辑
- 不要修改现有的 AI 对战逻辑

请先列出：
1. 准备修改的文件清单

完成后，等待确认再进入 Prompt 3。
```
