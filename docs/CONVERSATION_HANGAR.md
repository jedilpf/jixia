# 对话机库（长期协作记忆）

更新时间：2026-04-03

## 目标
- 记录我们最近协作中反复出现的问题。
- 固化必须遵守的执行顺序和沟通规则。
- 作为后续任务的快速对照清单，减少重复返工。

## 高频错误（已确认）
1. 在未完成策划确认前，直接进入实现或接入阶段。
2. 用“代码叠层遮挡”修补视觉，而不是输出完整独立资产图。
3. 只改一个入口链路，导致 `MainMenu` 与 `HomeScreen` 表现不一致。
4. 未先拆解现有设计体系（结构、符号、字体）就重画新按钮。
5. 图标语义复用（与“卡牌图鉴”主标识过于相似），造成入口混淆。
6. 字体风格与主按钮体系不统一，出现“看起来不是同一套”的问题。
7. 未先在对话框内给出可视结果确认，就继续下一阶段。

## 强制执行顺序（UI 设计任务）
1. Phase 1：策划与逆向拆解（不出最终图、不接入代码）。
2. Phase 2：草图方案（至少 A/B 两版），在对话框展示给你选。
3. Phase 3：按选中方案精修定稿（仍不接入）。
4. Phase 4：你确认定稿后，再接入到项目。
5. Phase 5：本地运行展示，收最终反馈。

## 你的固定偏好（必须遵守）
1. 先讲清“要改什么、为什么改、影响什么”，再动手。
2. 重要节点必须汇报，不允许闷头连续改完再汇总。
3. 支持并行 AI 协作场景：不随意删除“看起来多余”的文件。
4. 本地运行命令固定：`npx vite --host`。
5. 视觉任务要“同体系不重复”：结构一致、内容独立。
6. 指定文案必须准确：`争鸣史`（百家争鸣之争，历史之史）。

## 接入前自检（按钮类任务）
1. 是否是“一张完整按钮图”，而不是覆盖补丁方案？
2. 是否和另外三个按钮在框体、字体、光效上同体系？
3. 左侧符号是否与其他入口（特别是图鉴）语义明确区分？
4. 两条入口链路是否同步（`MainMenu` + `HomeScreen`）？
5. 是否已在对话框先展示图稿并获得你的确认？

## 变更纪律
- 未确认前：只允许做策划/草图，不接入主链路。
- 未通过时：只在当前阶段迭代，不跳阶段并行推进。
- 有冲突时：先仲裁记录，再实现。
