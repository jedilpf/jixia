**文档版本**: 2.0.0  
**更新日期**: 2026-03-09  
**适用范围**: 前端 / 服务端 / 策划配置 / QA / 数据

## 1. 统一基线

从 2026-03-09 起，开发口径统一为：

- `战斗机制总说明_v1`
- `首发4个论场完整设定文档_v1`
- 今日确认的 **170 张设定集**（含战斗设定）

任何旧术语、旧流程、旧示例如与上述冲突，直接废止。

## 2. 本地开发

### 2.1 环境

- Node.js >= 18
- npm >= 9

### 2.2 启动

```bash
npm install
npm run dev
```

### 2.3 质量命令

```bash
npm run typecheck
npm run build
```

---

## 3. 功能开发顺序（战斗链路 v2.0）

1. 回合状态机：`出牌 → 结算 → 出牌 → 结算`
2. 出牌后立即结算（即时反馈）
3. 双议区站位与伤害传递
4. 论场被动/偏向/论场技接入
5. 着书与成长接入
6. 对战日志、回放、战报与埋点

---

## 4. 配置开发规则（卡牌/论场/议题）

### 4.1 卡牌类型限定

仅允许五类：

- `立论`
- `策术`
- `反诘`
- `门客`
- `玄章`

### 4.2 新卡接入检查

- 是否与 170 张基线重复定位
- 是否至少对应一个明确回合窗口（明辩/暗谋/结算层）
- 是否有三席交互价值，不做“无位置感”卡
- 是否通过四论场兼容性检查

### 4.3 论场配置限定

每个论场只含三部分：

1. 基础被动
2. 偏向效果
3. 论场技（本局一次）

---

## 5. 测试清单（最小可回归）

- [ ] 出牌后立即结算
- [ ] 结算顺序正确且可复现
- [ ] 双议区传递正确（主议/旁议）
- [ ] 论场基础被动/偏向效果触发正确
- [ ] 论场技本局一次限制正确
- [ ] 着书带来可感知成长
- [ ] 回放与实时结算结果一致

---

## 6. 调试面板建议字段

- 对局：回合号、阶段、倒计时
- 双方资源：势、学识、护体、辩锋、失序（状态）
- 行动缓存：明辩提交、暗谋提交、锁定状态
- 结算日志：五层结算节点与结果
- 论场状态：被动、偏向、论场技是否已用

---

## 7. 禁止事项

- 禁止引入与当前基线冲突的旧牌类和旧回合模型
- 禁止跳过暗谋阶段做“伪同步”
- 禁止把三席当纯 UI，不进入真实结算
- 禁止把着书做成无成长反馈的伪弃牌
