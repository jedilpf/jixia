# 【任务说明文档】本地双人对战（"一起"）功能开发

> Author: Trae AI

## 一、任务背景

### 1.1 项目概述

《谋天下·问道百家》是一款以春秋战国诸子百家为题材的回合制卡牌对战游戏。

### 1.2 当前功能状态

| 功能 | 状态 | 说明 |
|------|------|------|
| AI对战 | ✅ 已实现 | 点击"开始对战"即可与AI对战 |
| **本地双人（"一起"）** | ❌ **不存在** | 项目中完全没有本地双人模式的代码 |
| 在线匹配 | ❌ 不存在 | 无网络对战功能 |

### 1.3 用户需求

用户希望在同一个设备上与家人朋友进行双人对战（本地"一起"玩），不需要网络连接。

---

## 二、具体要求

### 2.1 核心功能需求

1. **初始界面添加"一起"入口**
   - 在首页添加"一起玩"按钮
   - 与"开始对战（AI）"并列

2. **本地双人匹配流程**
   - 玩家1完成门派选择
   - 玩家2完成门派选择
   - 议题由双方协商或随机决定

3. **双人对战UI**
   - 左右分屏显示两位玩家
   - 玩家1操作区在左/下
   - 玩家2操作区在右/上
   - 回合同步显示

4. **回合切换机制**
   - 需要明确的"回合结束"按钮
   - 玩家1结束后切换到玩家2
   - 双方都结束视为回合完成

### 2.2 用户交互流程

```
首页
├── 开始对战（与AI） → PreBattleFlow（单人）
└── 一起玩（本地双人）
    ├── 玩家1选择门派
    ├── 玩家2选择门派
    ├── 议题确认
    └── 进入双人对战
        ├── 回合1：玩家1操作 → 玩家2操作
        ├── 回合2：玩家1操作 → 玩家2操作
        └── ...直到某方获胜
```

### 2.3 技术要求

- **不支持网络**：纯本地操作
- **分屏显示**：需要适配不同屏幕尺寸
- **输入设备**：键盘/鼠标切换
- **存档功能**：（可选）支持中途保存/加载

## 四、技术约束

### 4.1 技术栈

- React 18.2.0
- TypeScript 5.2.2
- Zustand 5.0.11
- Tailwind CSS

### 4.2 现有架构参考

```
src/
├── battleV2/
│   ├── engine.ts          # 战斗引擎
│   └── useDebateBattle.ts # 战斗Hook
├── components/
│   └── battle/
│       ├── SyncBattlePage.tsx    # 当前同步对战
│       ├── BattleBoardSync.tsx    # 当前战区
│       └── SyncBattleUI.tsx      # 当前UI
├── hooks/
│   └── useSyncBattle.ts  # 同步战斗Hook
└── utils/
    ├── syncBattleManager.ts
    ├── settlementEngine.ts
    └── phaseManager.ts
```

### 4.3 关键差异点

| 项目 | 当前AI对战 | 本地双人 |
|------|------------|----------|
| 输入源 | 单用户 | 双用户 |
| 状态管理 | 单一gameState | 分离playerState |
| 回合切换 | 自动 | 手动触发 |
| UI显示 | 单视角 | 分屏双视角 |
