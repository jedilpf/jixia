# 任务包（压缩版，可发给任意AI）

## 目标
把旧卡池迁移首批10张到 `content/cards/`，并保证 `npm run gate:daily` 通过。

## 只允许改这些路径
- `content/cards`
- `open_questions.md`

## 严格约束
1. 不允许新增锚点ID。映射不了就写 `UNKNOWN`，并登记到 `open_questions.md`。  
2. 每张卡必须包含：
   - `id,name,faction,rarity,type,cost,rules_text`
   - `keyword_ids,mechanic_ids,lore_anchor_ids,provenance,status`
3. 复杂度硬限制：
   - `rules_text` 字数 <= 25
   - 句数 <= 2
   - `keyword_ids` 数量 <= 2
4. 只能使用以下 mechanism id：
   - `MEC-CORE-DEFENSE`
   - `MEC-CORE-ORDER`
   - `MEC-CORE-COUNTER`
   - `MEC-CORE-DRAW`
   - `MEC-CORE-TEMPO`
5. `status` 统一写 `draft`。
6. `provenance` 必填：
   - `batch_id`
   - `models_used`
   - `canon_version`
   - `seed`

## 允许术语（keyword_ids）
- `护持`
- `怀疑`
- `清明`
- `反制`
- `净化`
- `论场`

## 输出要求
1. 输出10个卡牌JSON（文件名=卡牌ID.json）。  
2. 输出 `open_questions.md` 需要新增的条目（如果有 UNKNOWN）。  
3. 最后给出迁移报告：`card_id -> anchor/mechanic 映射依据`。

## 验收
- 本地必须可通过：`npm run gate:daily`
