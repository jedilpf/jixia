# CLAUDE Working Rules

This repository enforces a strict AI workflow:
- Start from `AI_START_HERE.md`
- Consume a task file from `ai/tasks/*.json`
- Use generated packet from `ai/out/*.prompt.md`
- Run `npm run ai:finalize -- --task <task-file>` before handoff

If any constraint conflicts with creative output:
- Constraints win
- Put unresolved items into `open_questions.md`

## 🎮 核心规则（必须严格遵守）

以下规则 **不能改、不能删、不能替代**：

### 1. 战场结构
- 每位玩家有 **2 个议区**：主议 + 旁议
- 每个议区最多放 **3 张牌**
- **不是**三席双层，**不是**6 个格子，就是 2 个议区

### 2. 胜利条件
- **先到 8 大势者获胜**
- 只有这一个胜利条件
- **没有**心证血量、**没有**证立胜利、**没有**双胜利条件

### 3. 回合流程（v2.0 简化版）
- **出牌 → 结算 → 出牌 → 结算**（两层结算机制）
- 简化流程，每轮出牌后立即结算
- 不再有明辩/暗谋的复杂等待

### 4. 卡牌类型
- **立论**：放在场上的单位（有辩锋 + 根基）
- **施策**：即时生效的法术
- **只有这两种**。不要加反诘、门客、玄章等额外类型

### 5. 资源（只有 6 个）
| 名称 | 说明 |
|------|------|
| 用度 | 出牌费用 |
| 辩锋 | 攻击力 |
| 根基 | 生命值 |
| 大势 | 积分，先到 8 赢 |
| 筹 | 减费用道具，最多 1 个 |
| 护印 | 护盾 |

**不要添加** 心证、灵势、证立、失序、文脉、机变等额外资源。

### 6. 每回合限制
- 最多出 **2 张牌**
- 起手 **5 张** 手牌

## ❌ 绝对不要做的事

1. 不要添加新的资源类型（心证、证立、失序、文脉、机变等）
2. 不要添加新的卡牌类型（反诘、门客、玄章等）
3. 不要添加新的议区（先声席、余论席等）
4. 不要添加第二胜利条件
5. 不要修改策划文档
6. 不要自己发明规则 — 文档没写的规则，等于不存在
7. 不要把简单问题复杂化 — 能用 if-else 解决的，不要搞设计模式
8. 不要删除已有功能除非明确要求

## ✅ 应该做的事

1. 读文档 — 每次改战斗逻辑前先读 `battle-design-核心规则-v1.2.md`

## 🔧 开发优先级（按顺序执行）

### Phase 1：最小可玩版本 ⬅️ 先做这个
**目标：能完成一场完整的单人 vs AI 对战**

[] 2 个议区（主议 + 旁议）战场布局
[] 明辩阶段：玩家选卡牌 → 放到议区
[] 暗谋阶段：玩家选卡牌 → 扣下
[] 揭示阶段：双方翻牌
[] 结算阶段：比辩锋，赢方 + 大势或 + 筹
[] AI 能自动出牌（最简单的策略就行）
[] 先到 8 大势显示胜利
[] 结果页：返回主菜单 / 再来一次
[] 10-20 张卡牌能用

### Phase 2：完善体验
[] 完整用度曲线
[] 完整手牌管理（抽牌 / 弃牌）
[] 4 个论场效果
[] 议题系统
[] UI 风格统一（青铜风格）
[] 新手引导

### Phase 3：内容填充
[] 170 张卡牌效果全实现
[] 故事模式
[] 着书系统
[] 社区功能

**严格按 Phase 1 → Phase 2 → Phase 3 顺序执行。**
