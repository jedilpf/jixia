# Prototype Script (English) - Vertical Slice Protocol

## What This Prototype Must Prove
In 3 rounds, the audience should understand:
1. This is a media design project, not only a game feature demo.
2. Philosophical identity changes strategic behavior.
3. Cultural framing and tactical readability can coexist.

## Locked Prototype Scope (Non-Negotiable)
- Schools: **2** (礼心殿, 玄匠盟)
- Topic: **1** ("Should virtue lead law, or should law lead virtue?")
- Battleground: **1** (Jixia Standard Debate Court)
- Card pool: **12-20** cards (recommended: **16**)
- Rounds: **3** complete rounds only

## Presentation Flow (8-12 Minutes)
1. **Framing (1 min)**
   - Say the core research question in one sentence.
   - Explain that the demo is a strict vertical slice for evaluation.
2. **Round 1: Identity in Action (2-3 min)**
   - Show first card decisions from each school.
   - Explain how each choice reflects a school-specific logic.
3. **Round 2: Tactical Translation (2-3 min)**
   - Show adaptation to board pressure.
   - Point out the design tradeoff: expression vs efficiency.
4. **Round 3: Closure and Interpretation (2-3 min)**
   - Finish the full loop and resolve outcome.
   - Explain the winner through argument structure, not only numbers.
5. **Reviewer Debrief (1-2 min)**
   - What was readable?
   - What felt culturally grounded?
   - What would scale next?

## Aesthetic Checkpoints During Demo
- Does the interface maintain debate atmosphere while keeping decision clarity?
- Do transitions feel like rhetorical progression, not random turn switching?
- Can observers infer faction identity from play patterns without extra lore explanation?

## Presenter Guardrails
- Keep language value-first and decision-first.
- Do not expand scope mid-demo (no extra schools/topics/rounds).
- If a supporting artifact is missing, mark it as `UNKNOWN` and continue the protocol.
